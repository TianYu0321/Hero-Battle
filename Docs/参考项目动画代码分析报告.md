## 一、Friday Night Funkin' (Psych Engine) — 角色动画与舞台演出

### 1.1 角色数据驱动架构 (`Character.hx`)

FNF 的核心设计是**JSON 配置文件驱动角色**。每个角色对应一个 `characters/xxx.json`：

```haxe
typedef CharacterFile = {
  var animations:Array<AnimArray>;  // 动画列表
  var image:String;                 // 图集路径
  var scale:Float;                  // 全局缩放
  var sing_duration:Float;          // 唱歌姿势保持时长倍数
  var position:Array<Float>;        // 场景基准偏移
  var camera_position:Array<Float>; // 相机焦点偏移
  var flip_x:Bool;
  var healthbar_colors:Array<Int>;  // 血条颜色 RGB
}

typedef AnimArray = {
  var anim:String;      // 逻辑名（如 "singLEFT"）
  var name:String;      // 图集内帧标签名
  var fps:Int;
  var loop:Bool;
  var indices:Array<Int>; // 指定帧序列（可选）
  var offsets:Array<Float>; // 该动画的 x,y 偏移
}
```

**借鉴点**：
- **分离数据与代码**：角色外观、动画、偏移全部外置 JSON，不用改代码就能添加新角色。
- **动画偏移表 (`animOffsets`)**：每个动画有独立的 `offset.x/y`，解决不同姿势中心点对不齐的问题。Godot 中可用 `AnimatedSprite2D` 的 `offset` 或父节点 `Node2D` 的 `position` 模拟。
- **统一播放接口 `playAnim(name, force, reversed, frame)`**：所有状态切换走同一个入口，内部处理 Atlas 播放和偏移应用。

### 1.2 动画状态机 (`Character.update()`)

```haxe
// 核心状态变量
var holdTimer:Float = 0;      // 唱歌姿势计时器
var heyTimer:Float = 0;       // "Hey!" 姿势计时器
var specialAnim:Bool = false; // 标记特殊动画（如被击、欢呼）
var danceIdle:Bool = false;   // 是否用 danceLeft/danceRight 代替 idle

// update() 中的自动状态回退
if (heyTimer > 0) {
  heyTimer -= elapsed;
  if (heyTimer <= 0 && specialAnim) { specialAnim = false; dance(); }
}
else if (specialAnim && isAnimationFinished()) {
  specialAnim = false; dance();
}
else if (getAnimationName().endsWith('miss') && isAnimationFinished()) {
  dance(); // miss 动画播完回 idle
}

// 非玩家角色：holdTimer 超阈值自动回 idle
if (!isPlayer && holdTimer >= Conductor.stepCrochet * 0.0011 * singDuration) {
  dance(); holdTimer = 0;
}

// 循环动画支持：播放完自动接 loop 版本
if (isAnimationFinished() && hasAnimation('$name-loop'))
  playAnim('$name-loop');
```

**借鉴点**：
- **自动回退机制**：任何临时姿势（攻击、受击、说话）都有计时器，超时自动回到 `dance()`/`idle()`。这正适合我们的回合制战斗——角色攻击/受击后自动回到待机。
- **`specialAnim` 锁**：特殊动画期间禁止 `dance()` 打断，避免状态冲突。
- **Loop 后缀约定**：`attack` → `attack-loop` 的自动衔接，适合长持续动作。

### 1.3 节拍同步舞蹈 (`dance()`)

```haxe
public function dance() {
  if (!debugMode && !skipDance && !specialAnim) {
    if (danceIdle) {
      danced = !danced;
      playAnim(danced ? 'danceRight' + idleSuffix : 'danceLeft' + idleSuffix);
    } else if (hasAnimation('idle' + idleSuffix)) {
      playAnim('idle' + idleSuffix);
    }
  }
}
```

**借鉴点**：
- **idleSuffix**：可以切换不同 idle 变体（如 `"-alt"`），适合季节/皮肤变化。
- **danceIdle 左右交替**：GF 类型的角色用左右摇摆代替静止 idle，更有生气。我们的 Q 版角色也可以用呼吸/摇摆 idle。

### 1.5 相机系统 (`PlayState.hx`)

```haxe
// 相机跟随目标
camFollow = new FlxObject();
camFollow.setPosition(camPos.x, camPos.y);
FlxG.camera.follow(camFollow, LOCKON, 0);
FlxG.camera.zoom = defaultCamZoom;

// 节拍缩放回弹
if (camZooming) {
  FlxG.camera.zoom = FlxMath.lerp(defaultCamZoom, FlxG.camera.zoom, Math.exp(-elapsed * 3.125 * camZoomingDecay));
}

// 相机速度系数
cameraSpeed:Float = 1;
FlxG.camera.followLerp = 0.04 * cameraSpeed * playbackRate;
```

**借鉴点**：
- **相机焦点对象 `camFollow`**：不直接移动相机，而是移动一个隐形目标点，相机平滑跟随。Godot 中可用 `Marker2D` + `Camera2D` 的 `position_smoothing` 实现。
- **节拍缩放**：重拍时 `zoom += 0.015`，然后每帧 lerp 回 `defaultCamZoom`，产生节奏感。非常适合战斗中的大招演出。

## 二、FNF 对话系统 (`DialogueBoxPsych.hx` + `DialogueCharacter.hx`)

  var sound:String;        // 打字音效

### 2.2 角色立绘系统

```haxe
function spawnCharacters() {
  // 1. 先扫描整个对话，收集所有出现的角色（去重）
  // 2. 预实例化所有角色立绘，初始 alpha=0.00001（隐藏）
  // 3. 左/中/右三种位置配置
  
  var char:DialogueCharacter = new DialogueCharacter(x, y, individualChar);
  char.alpha = 0.00001; // 隐藏
  add(char);
}

// 切换对话时：当前说话角色滑入，其他滑出
if (i != lastCharacter) {
  // 非当前角色：滑出 + 淡出
  char.x -= scrollSpeed * elapsed;
  char.alpha -= 3 * elapsed;
} else {
  // 当前角色：滑入 + 淡入
  char.x += scrollSpeed * elapsed;
  char.alpha += 3 * elapsed;
}
```

**借鉴点**：
- **预加载 + 池化**：对话开始前扫描全部角色并预加载，避免切换时卡顿。
- **滑入滑出动画**：角色不是瞬间出现，而是以 `scrollSpeed` 平移 + alpha 渐变。适合我们的剧情演出。
- **气泡状态机**：`normalOpen` → `normal` / `angryOpen` → `angry`，带开/关动画。

### 2.3 对话角色动画 (`DialogueCharacter.hx`)

```haxe
// 每个表情有两个动画：
// - "talk" : 说话时的 loop 动画
// - "talk-IDLE" : 不说话时的 idle 动画

typedef DialogueAnimArray = {
  var anim:String;           // 逻辑名
  var loop_name:String;      // 说话动画（如 "talk"）
  var loop_offsets:Array;    // 说话动画偏移
  var idle_name:String;      //  idle 动画（如 "talk-IDLE"）
  var idle_offsets:Array;    // idle 动画偏移
}

public function playAnim(animName, playIdle=false) {
  animation.play(playIdle ? leAnim + IDLE_POSTFIX : leAnim, false);
  // 应用对应偏移
  offset.set(playIdle ? anim.idle_offsets : anim.loop_offsets);
}
```

**借鉴点**：
- **说话/Idle 双动画**：角色说话时播放 loop 口型动画，说完自动切到 idle。我们的剧情对话可以直接借鉴。
- **独立偏移**：loop 和 idle 的偏移分开配置，解决换动画时的位置跳动。

## 三、Phantom Camera — 镜头系统 (`phantom_camera_2d.gd` + `phantom_camera_host.gd`)

### 3.1 核心架构

```
Camera2D (场景相机)
  └── PhantomCameraHost (子节点，接管相机)
        └── 监听所有 PhantomCamera2D 的 priority 变化

Scene 中散布多个 PhantomCamera2D（虚拟相机）
  └── 每个有 priority、follow_mode、tween_resource
```

### 3.2 Priority 切换系统

```gdscript
# PhantomCameraHost 的核心逻辑
func _check_pcam_priority(pcam: Node) -> void:
  if pcam.get_priority() >= _active_pcam_priority:
    _assign_new_active_pcam(pcam)

func _assign_new_active_pcam(pcam: Node) -> void:
  # 1. 记录旧相机状态
  _prev_active_pcam_2d_transform = camera_2d.global_transform
  _camera_zoom = camera_2d.zoom
  
  # 2. 激活新相机
  _active_pcam_2d = pcam
  _active_pcam_priority = pcam.priority
  
  # 3. 触发 Tween 过渡
  _trigger_pcam_tween = true
  _tween_elapsed_time = 0.0
```

**借鉴点**：
- **Priority 竞争**：不需要手动切相机，只需改变 `priority` 值，系统自动处理过渡。我们的战斗可以在「角色特写」「全景」「UI 面板」之间优雅切换。
- **自动 Tween**：切换时自动记录旧状态，向新状态插值。

### 3.3 Tween 插值 (`_pcam_tween()`)

```gdscript
func _pcam_tween(delta: float) -> void:
  _tween_elapsed_time = min(_tween_duration, _tween_elapsed_time + delta)
  
  # 位置插值
  camera_2d.global_position = _tween_interpolate_value(
    _prev_active_pcam_2d_transform.origin,
    _active_pcam_2d_glob_transform.origin
  )
  
  # 旋转插值
  camera_2d.rotation = _tween_interpolate_value(...)
  
  # 缩放插值
  camera_2d.zoom = _tween_interpolate_value(_camera_zoom, _active_pcam_2d.zoom)

func _tween_interpolate_value(from: Variant, to: Variant) -> Variant:
  return Tween.interpolate_value(from, to - from, _tween_elapsed_time, _tween_duration, _tween_transition, _tween_ease)
```

**借鉴点**：
- **统一插值入口**：用 `Tween.interpolate_value()` 处理所有属性（位置、旋转、缩放），避免多个 Tween 节点混乱。
- **Transform 级别过渡**：不只是位置，旋转和缩放也一起过渡，保证镜头运动感完整。

### 3.4 SmoothDamp 算法 (`_smooth_damp()`)

```gdscript
func _smooth_damp(target_axis, self_axis, index, current_velocity, set_velocity, damping_time, delta):
  damping_time = maxf(0.0001, damping_time)
  var omega: float = 2 / damping_time
  var x: float = omega * delta
  var exponential: float = 1 / (1 + x + 0.48 * x * x + 0.235 * x * x * x)
  var diff: float = self_axis - target_axis
  
  var temp: float = (current_velocity + omega * diff) * delta
  set_velocity.call(index, (current_velocity - omega * temp) * exponential)
  var output: float = target_axis + (diff + temp) * exponential
  
  # 防止 overshoot
  if (target_axis - self_axis > 0.0) == (output > target_axis):
    output = target_axis
  return output
```

**借鉴点**：
- **SmoothDamp 比 Lerp 更自然**：Lerp 永远到不了目标，SmoothDamp 有明确的到达时间和速度衰减曲线，适合相机跟随。
- **速度缓存**：`_follow_velocity_ref` 记录当前速度，下一帧继续用，保证连续性。

### 3.5 其他实用模式

| 模式 | 用途 | 我们游戏中的场景 |
|------|------|----------------|
| **GLUED** | 相机完全贴死目标 | 不需要 |
| **SIMPLE** | 平滑跟随 + 偏移 | 日常探索 |
| **GROUP** | 框选多个目标，自动缩放 | 战斗双方同框 |
| **FRAMED** | 死区跟随（目标在死区内相机不动） | 角色在小范围内移动 |
| **Lookahead** | 根据目标速度预测相机位置 | 角色冲刺时提前偏移 |
| **Noise** | 持续震动/噪声 | 受击、大招、地震 |

## 四、Godot Cutout Helper — 纸片人动画

### 4.1 节点架构

```
BodyPartRoot (自定义节点)
  ├── BodyPartNode (头)
  │     └── Sprite2D / AnimatedSprite2D
  ├── BodyPartNode (身体)
  │     └── Sprite2D
  ├── BodyPartNode (左手)
  │     └── Sprite2D
  └── ...
```

### 4.2 Action 系统

- **BodyPartRoot** 维护一个 Action 列表，每个 Action 对应一套姿势。
- **BodyPartNode** 每个部位维护多套纹理，按 Action 分组。
- **运行时切换**：`$BodyPartRoot.setNewSprites("attack", 0)` 一键切换所有部位到「攻击」姿势的纹理。

**借鉴点**：
- **纹理换装，不动骨骼**：适合我们的赛璐璐 Q 版——每个姿势画一套完整的部位图，切换时换图 + 微调旋转。
- **与 Skeleton2D 结合**：可以用 Godot 原生的 `Skeleton2D` + `Bone2D` + `RemoteTransform2D` 做骨骼，Cutout Helper 做纹理管理。
- **编辑器预览**：底部面板切换 Action，方便美术调整。

### 4.3 Godot 原生 Cutout 方案（官方文档）

```
Node2D (Root)
  ├── Sprite2D (Hip)
  │     ├── Sprite2D (Torso)
  │     │     ├── Sprite2D (Head)
  │     │     ├── Sprite2D (Arm_R)
  │     │     │     ├── Sprite2D (Forearm_R)
  │     │     │     │     └── Sprite2D (Hand_R)
  │     │     └── Sprite2D (Arm_L)  [RemoteTransform2D 修正深度]
  │     └── Sprite2D (Leg_R)...
```

- 用 `RemoteTransform2D` 解决「子节点必须在父节点前面」的问题（比如左手要在身体后面）。
- 官方 **Skeleton2D** 支持 IK（编辑器内），运行时可用 GDScript 实现简单 IK。

## 五、综合借鉴方案：Godot 4 实现路线

### 5.1 角色动画系统（受 FNF 启发）

```gdscript
# scripts/actors/actor_controller.gd
class_name ActorController
extends Node2D

@export var actor_data: ActorData  # Resource，对应 FNF 的 JSON

@onready var sprite: AnimatedSprite2D = $AnimatedSprite2D

var _hold_timer: float = 0.0
var _special_anim: bool = false
var _anim_offsets: Dictionary = {}  # anim_name -> Vector2

func play_anim(anim_name: String, force: bool = false) -> void:
    _special_anim = false
    sprite.play(anim_name)
    if _anim_offsets.has(anim_name):
        sprite.offset = _anim_offsets[anim_name]

func play_special(anim_name: String, duration: float) -> void:
    # 特殊动画（受击、放大招），期间不自动回 idle
    _special_anim = true
    play_anim(anim_name)
    await get_tree().create_timer(duration).timeout
    _special_anim = false
    _return_to_idle()

func _return_to_idle() -> void:
    if _special_anim: return
    play_anim("idle")

func _process(delta: float) -> void:
    # 非 idle 动画计时，超时回 idle
    if not _special_anim and sprite.animation != "idle":
        _hold_timer += delta
        if _hold_timer >= actor_data.idle_return_time:
            _hold_timer = 0
            _return_to_idle()
    else:
        _hold_timer = 0
```

### 5.3 对话系统（受 FNF DialogueBoxPsych 启发）

```gdscript
# scripts/ui/dialogue_system.gd
class_name DialogueSystem
extends Control

@export var dialogue_data: DialogueData
@export var text_speed: float = 0.05

var _current_line: int = 0
var _is_typing: bool = false
var _character_slots: Dictionary = {}  # portrait_name -> DialogueCharacter

@onready var text_label: RichTextLabel = $TextLabel
@onready var bubble: AnimatedSprite2D = $Bubble

func start_dialogue(data: DialogueData) -> void:
    dialogue_data = data
    _current_line = 0
    _preload_characters()
    _show_line()

func _preload_characters() -> void:
    # 扫描全部对话行，预加载所有出现的角色
    var needed: Array[String] = []
    for line in dialogue_data.lines:
        if not needed.has(line.portrait):
            needed.append(line.portrait)
    
    for char_name in needed:
        var char_scene = load("res://scenes/dialogue/characters/%s.tscn" % char_name)
        var char = char_scene.instantiate()
        _character_slots[char_name] = char
        $CharacterContainer.add_child(char)
        char.modulate.a = 0.0  # 初始隐藏

func _show_line() -> void:
    var line = dialogue_data.lines[_current_line]
    
    # 切换当前角色显隐
    for name in _character_slots:
        var char = _character_slots[name]
        var target_alpha = 1.0 if name == line.portrait else 0.0
        create_tween().tween_property(char, "modulate:a", target_alpha, 0.2)
        # 滑入/滑出
        var target_x = char.base_position.x if name == line.portrait else char.base_position.x - 100
        create_tween().tween_property(char, "position:x", target_x, 0.3)
    
    # 播放角色表情
    if _character_slots.has(line.portrait):
        _character_slots[line.portrait].play_expression(line.expression)
    
    # 打字机效果
    _type_text(line.text, line.speed if line.speed > 0 else text_speed)
    
    # 切换气泡状态（normal/angry/shout）
    bubble.play(line.bubble_state + "_open")
    await bubble.animation_finished
    bubble.play(line.bubble_state)

func _type_text(text: String, speed: float) -> void:
    text_label.text = ""
    _is_typing = true
    for i in text.length():
        text_label.text += text[i]
        await get_tree().create_timer(speed).timeout
        # 播放打字音效
        AudioManager.play_ui("dialogue_tick")
    _is_typing = false
```

### 5.4 战斗镜头系统（受 Phantom Camera 启发）

由于 Phantom Camera 是一个完整插件，我们的项目可以直接安装使用。但如果想自己实现轻量版：

```gdscript
# scripts/camera/battle_camera.gd
class_name BattleCamera
extends Camera2D

@export var default_zoom: float = 1.0
@export var zoom_smoothing: float = 5.0
@export var position_smoothing_speed: float = 4.0

var _target_position: Vector2
var _target_zoom: float = 1.0
var _shake_strength: float = 0.0

func focus_on(target: Node2D, zoom_level: float = 1.2, duration: float = 0.5) -> void:
    _target_position = target.global_position
    _target_zoom = zoom_level
    # Tween 动画
    var tween = create_tween().set_trans(Tween.TRANS_CUBIC).set_ease(Tween.EASE_IN_OUT)
    tween.tween_property(self, "_target_zoom", zoom_level, duration)

func shake(strength: float, duration: float) -> void:
    _shake_strength = strength
    var tween = create_tween()
    tween.tween_property(self, "_shake_strength", 0.0, duration)

func _process(delta: float) -> void:
    # 平滑跟随目标
    global_position = global_position.lerp(_target_position, 1.0 - exp(-position_smoothing_speed * delta))
    zoom = zoom.lerp(Vector2.ONE * _target_zoom, 1.0 - exp(-zoom_smoothing * delta))
    
    # 震动
    if _shake_strength > 0:
        offset = Vector2(randf_range(-1, 1), randf_range(-1, 1)) * _shake_strength
    else:
        offset = Vector2.ZERO
```

### 5.5 纸片人战斗角色（Cutout 方案）

**方案 A：纯 Sprite 切换（最简单，适合 Q 版）**
- 每个角色一个 `AnimatedSprite2D`
- 每个姿势（idle/attack/hit/dead）一套完整的 SpriteFrames
- 代码中直接 `sprite.play("attack")`

**方案 B：部位拼接 + 骨骼（适合需要物理/动态效果）**
```
Node2D (ActorRoot)
  ├── Sprite2D (BackArm)      [z_index -1]
  ├── Sprite2D (Body)
  ├── Sprite2D (Head)
  │     └── Sprite2D (Face)   [表情切换]
  ├── Sprite2D (FrontArm)
  └── Sprite2D (Weapon)
```
- 用 `RemoteTransform2D` 把部位挂到 `Skeleton2D` 骨骼上
- 用 AnimationPlayer 做关键帧动画（旋转、位移）
- 用 `BodyPartRoot/BodyPartNode` 思路管理多套纹理（攻击套、受击套）

**推荐**：我们的项目目前规模不大，**方案 A（完整 SpriteFrames）** 是最佳起点。等角色数量多了再迁移到方案 B。
