## 问题 3：战斗面板两套卡片节点 — 需清理合并 ⚠️

**现状**：`battle_animation_panel.tscn` 中有两套 HeroCard/EnemyCard：

```
HudContainer/HeroCard     ← 旧的（ColorRect Portrait + NameLabel + HpBar + HpMeta）
StageArea/HeroCard          ← 新的（TextureRect Portrait + PortraitOverlay + GlowOverlay）

HudContainer/EnemyCard     ← 旧的
StageArea/EnemyCard        ← 新的
```

**脚本引用**：
```gdscript
@onready var hero_portrait_hud: ColorRect = $HudContainer/HeroCard/Portrait  ## 旧的
@onready var hero_portrait: TextureRect = $StageArea/HeroCard/Portrait       ## 新的
```

**风险**：
- 两套卡片同时在画面中会视觉重叠（玩家看到两个头像框）
- `HudContainer` 下的旧卡片是 ColorRect（纯色块占位），不是真正的角色头像
- 脚本同时维护两套引用，容易混淆

**建议处理方案**：

**方案 A（推荐）**：删除 `HudContainer` 下的 HeroCard/EnemyCard，把 NameLabel/HpBar/HpMeta 移到 `StageArea` 的卡牌内或独立节点。所有战斗动画只操作 `StageArea` 的卡牌。

**方案 B**：保留 `HudContainer` 作为"信息面板"（只显示名字+HP条，不显示头像），`StageArea` 作为"战斗动画卡片"（只显示头像+特效）。需要确保两者位置不重叠。

**当前 run_main 的伙伴卡片化也用了 HudContainer**（`HudContainer/PartnerCardSlot_x`），这是正确的——伙伴卡片是信息展示，和战斗动画无关。

**具体操作建议**：
1. 检查 `HudContainer/HeroCard` 和 `StageArea/HeroCard` 在运行时是否同时可见
2. 如果 `HudContainer/HeroCard` 在战斗动画时也被设为 visible=true，就会出现重叠
3. 最简单的修复：把 `HudContainer/HeroCard` 和 `HudContainer/EnemyCard` 从 `.tscn` 中删除，或把它们的 `visible` 默认设为 `false`
4. 确保 `_update_hp_display()` 引用的 `hero_hp_bar` 和 `enemy_hp_bar` 指向正确的节点（当前指向 `HudContainer`，需要确认数值显示是否正常）
