## 1. 核心风格命名

本项目整体美术风格定义为：

> **日系赛璐璐 SD 纸片小剧场风格**

也可以扩展描述为：

> **以日系动画赛璐璐上色为基础，结合 SD/Q版角色比例、完整纸片 Pose 翻面式动画、漫画拟声词反馈、2D 动漫舞台背景，以及明亮卡片式 UI 的轻量养成乱斗风格。**

内部简称：

> **SD 纸片小剧场风**

---

## 2. 一句话风格定位

> 本游戏采用“日系赛璐璐 SD 纸片小剧场风格”：角色以完整纸片 Pose 的方式进行翻面、弹跳、倾斜、换姿势和自动乱斗，背景采用 2D 动漫舞台布景，UI 使用明亮卡片与贴纸式动效，整体呈现像动漫小剧场一样轻快、可爱、热血但不严肃的养成战斗体验。

---

## 3. 整体视觉目标

游戏整体给人的第一感觉应该是：

> **像一群可爱的纸片小角色，在一个动画小剧场里训练、冒险、乱斗和成长。**

它不是写实 3D，也不是厚重奇幻 RPG。  
它应该是轻快、明亮、可爱、有角色魅力、有舞台感的 2D 游戏。

### 3.1 核心关键词

- 日系动画感
- 赛璐璐上色
- SD/Q版比例
- 纸片人 Pose 演出
- 小剧场舞台感
- 漫画拟声词反馈
- 自动乱斗观赏性
- 养成感
- 轻奇幻
- 可爱但不幼稚
- 热血但不严肃
- 搞笑但不廉价

## 4. 角色美术风格

### 4.1 角色比例

角色采用 **SD/Q版比例**，建议控制在：

> **2.5 头身到 3 头身**

不建议低于 2 头身，否则容易过度幼儿化、表情包化。  
不建议超过 4 头身，否则会削弱轻松乱斗感。

### 4.2 推荐比例标准

以 3 头身角色为基准：

| 部位 | 推荐比例 |
|---|---|
| 头部高度 | 角色总高的 1/3 左右 |
| 身体高度 | 角色总高的 1/3 左右 |
| 腿部高度 | 角色总高的 1/3 左右 |
| 头宽 | 略大于肩宽 |
| 肩宽 | 约为头宽的 75%-90% |
| 手脚 | 简化，但要保留动作识别度 |
| 武器/道具 | 可适度夸张，增强剪影 |

### 4.3 职业差异建议

不同职业可以通过比例微调体现差异：

| 职业方向 | 视觉调整 |
|---|---|
| 战士 / 坦克 | 躯干更宽，武器更大，站姿更稳 |
| 刺客 / 游侠 | 身体略瘦，动作更倾斜，剪影更尖锐 |
| 法师 / 牧师 | 身体略小，道具、帽子、飘带更明显 |
| 辅助 / 商人 | 道具感更强，表情更丰富 |
| 怪物 / 敌人 | 轮廓更夸张，但仍保持 Q版可读性 |

## 5. 角色上色与线条规范

### 5.1 上色方式

角色采用：

> **日系赛璐璐上色**

要求：

- 色块干净
- 阴影边界明确
- 不做复杂厚涂
- 不做写实皮肤质感
- 高光简洁
- 颜色明快
- 边缘线清晰
- 角色整体像“可以动起来的动画纸片”

### 5.2 阴影层级

建议控制为：

| 阴影层级 | 用途 |
|---|---|
| 基础色 | 角色主色块 |
| 一层阴影 | 主要体积与遮挡 |
| 二层阴影 | 仅用于头发、披风、武器等重点部件 |
| 高光 | 少量使用，避免油腻 |

不建议使用过多渐变。  
渐变可以存在，但不应超过角色视觉面积的 15%-20%。

### 5.3 线条规范

线条应保持：

- 干净
- 圆润
- 稍有手绘感
- 外轮廓略粗
- 内部细节适当简化
- 不要机械硬边
- 不要写实复杂线稿

建议：

| 线条类型 | 方向 |
|---|---|
| 外轮廓线 | 略粗，帮助角色从背景中分离 |
| 内部结构线 | 较细，避免角色显脏 |
| 表情线 | 清晰、夸张、可读 |
| 武器线 | 稍微锐利，但不能破坏可爱感 |

## 6. 色彩规范

### 6.1 角色、背景、UI 的色彩关系

整体色彩目标：

> **角色突出，背景退后，UI 清晰。**

| 元素 | 色彩倾向 |
|---|---|
| 角色 | 高明度 / 中高饱和 |
| 背景 | 中明度 / 中低饱和 |
| UI | 高明度 / 中饱和 / 强可读性 |
| 特效 | 短时间高亮，但不能长期刺眼 |
| 敌人 | 可比角色略暗或略冷，但不能糊成一团 |

### 6.2 色彩使用原则

- 角色主色要明确。
- 每个角色建议有 1 个主色、1-2 个辅色、1 个点缀色。
- 可以参考 60:30:10 的配色比例：
  - 60% 主色
  - 30% 辅色
  - 10% 点缀色
- 背景不要与角色主色过度接近。
- 角色与背景之间应有明显明度差。
- 重要角色可以加轻微白边或浅色边缘光，增强舞台贴纸感。

### 6.3 禁止色彩方向

不建议：

- 大面积荧光色
- 过暗低对比色
- 脏灰色主导画面
- 暗黑 RPG 式压抑色调
- 过度赛博朋克霓虹色
- 真实油画式复杂色彩混合

## 7. 光影规范

### 7.1 统一光源

默认采用：

> **左上方主光源**

适用于角色、UI 贴纸、部分场景物件。

### 7.2 光影风格

角色光影应保持：

- 简洁
- 赛璐璐
- 边界明确
- 不追求真实体积
- 不使用复杂环境光
- 不做写实反射

### 7.3 角色落地阴影

角色脚下建议添加：

> **简单椭圆形阴影**

作用：

- 让纸片角色站在舞台上
- 增强空间定位
- 避免角色像漂浮贴图

阴影要求：

- 半透明
- 形状简单
- 不要写实投影
- 不要根据复杂光照变化

### 8.1 基础表情类型

每个主要角色建议至少准备：

- 普通
- 开心
- 生气
- 惊讶
- 受击
- 得意
- 紧张

### 8.2 表情夸张程度

允许：

- 夸张眉毛
- 星星眼

## 9. 动画表现核心定义

### 9.1 最重要的修正

本项目角色动画：

> **保持 60fps 的运行与播放标准。**

纸片感 **不是通过降低帧率实现**。  
纸片感来自以下动作语言：

1. **翻面切 Pose**
2. **拟声词 / 音效显示**
3. **整张角色模型的方向倾斜**
4. **位移、缩放、旋转补间**
5. **命中停顿、镜头反馈、弹性运动**

因此，本项目动画本质是：

> **60fps 下的纸片人 Pose 演出风格。**

---

## 10. 角色动画制作方式

### 10.1 不做复杂部位拆分

初版角色动画不采用：

- 复杂 Skeleton2D 骨骼
- 头、手、脚、身体分开驱动
- Spine 式复杂骨骼动画
- 高成本逐帧手绘动作

### 10.2 采用完整 Pose 图

角色动画采用：

> **完整角色 Pose 图 + 翻面过渡 + 整体补间**

每张 Pose 图都是一个完整角色，不拆成头、身体、手臂、腿。

例如：

```text
hero_idle.png
hero_attack_start.png
hero_attack_hit.png
hero_hit.png
hero_victory.png
```

### 10.3 动作构成三层

角色动作由三层组成：

#### A. Pose 层

用于表现动作状态变化：

- 待机 Pose
- 攻击 Pose
- 受击 Pose
- 技能 Pose
- 胜利 Pose
- 失败 Pose

#### B. 变换层

通过整个模型的变换制造动作：

- 整体位移
- 整体旋转
- 整体缩放
- 翻面过渡
- 回弹
- 压缩
- 倾斜

#### C. 反馈层

通过外部反馈强化打击感：

- 拟声词飞出
- 伤害数字
- 速度线
- 闪白
- 镜头轻震
- 命中停顿
- UI 气泡提示
- 音效

---

## 11. 纸片感的核心动作语言

### 11.1 翻面切 Pose

纸片感最关键的来源是：

> **角色像一张纸片立牌一样，翻一下面后切到另一个动作 Pose。**

示例：

```text
待机.png
→ 翻面过渡
→ 攻击.png
```

视觉表现：

- 角色横向压缩
- 短暂接近一条线
- 切换动作 Pose
- 再展开
- 像纸片被拨动或翻面

### 11.2 拟声词 / 音效显示

攻击、命中、暴击、支援登场时，应出现漫画式拟声词。

示例：

- “啪！”
- “砰！”
- “斩！”
- “咚！”
- “嗖！”
- “Duang!”
- “哐！”

作用：

- 强化漫画感
- 强化打击反馈
- 弥补不拆分部位带来的动作细节不足
- 增强自动战斗的观赏性
- 形成小剧场看戏感

### 11.3 整张模型方向调整

动作不是靠某个手臂单独挥动，而是：

> **整张角色 Pose 随动作方向发生倾斜和位移。**

示例：

| 动作 | 模型表现 |
|---|---|
| 向下横劈 | 整个模型向劈砍方向倾斜，下压 |
| 向上挑击 | 整个模型向上抬起，后仰再前推 |
| 前冲刺击 | 整个模型向前倾，快速位移 |
| 受击 | 整个模型后仰、抖动、退后 |
| 支援登场 | 整个模型从画面边缘滑入并弹跳停住 |

## 12. 标准攻击动作示例

以“向下横劈”为例。

### 12.1 演出流程

1. **待机 Pose**
   - 角色轻微上下浮动
   - 模型保持正常朝向

2. **前摇**
   - 模型轻微后撤
   - 身体向攻击反方向蓄力
   - 轻微压缩

3. **翻面过渡**
   - 角色横向快速压缩
   - 切换到攻击 Pose

4. **攻击 Pose**
   - 模型整体向下横劈方向倾斜
   - 同时向前位移
   - 出现速度线或挥砍轨迹

5. **命中反馈**
   - 敌人后仰
   - 敌人闪白
   - 拟声词“斩！”飞出
   - 伤害数字弹出
   - 镜头轻震
   - 命中停顿 0.05-0.12 秒

6. **收招**
   - 模型回弹
   - 角色恢复待机 Pose
   - 继续待机循环

### 12.2 视觉特点

- 动作阶段清晰
- 纸片感明显
- 打击感来自整体运动与反馈层
- 不依赖复杂骨骼
- 60fps 保证播放流畅

## 13. 基础动作清单

每个主要角色建议至少准备以下动作：

| 动作 | 说明 |
|---|---|
| idle | 待机 |
| attack | 普通攻击 |
| skill | 技能 / 大招 |
| hit | 受击 |
| support_in | 支援登场 |
| support_action | 支援动作 |
| victory | 胜利 |
| defeat | 失败 / 倒地 |

### 13.1 每个动作推荐 Pose 数

| 动作 | 推荐 Pose 数 |
|---|---:|
| 待机 | 1-3 张 |
| 普通攻击 | 2-4 张 |
| 受击 | 1-2 张 |
| 技能 | 3-6 张 |
| 支援登场 | 1-3 张 |
| 胜利 | 2-4 张 |
| 失败 | 1-2 张 |

注意：  
Pose 数不是帧率。  
动画播放仍然是 60fps，Pose 之间通过翻面、补间、倾斜、缩放和特效进行连接。

## 14. Godot 初版实现建议

### 14.1 推荐技术组合

初版优先使用：

```text
Sprite2D
AnimationPlayer
Tween
Camera2D
Label / RichTextLabel
CanvasLayer
```

可选：

```text
AnimatedSprite2D
SpriteFrames
ShaderMaterial
GPUParticles2D
```

### 14.2 不优先使用

初版不建议优先使用：

```text
Skeleton2D
复杂骨骼动画
Spine / DragonBones
部件分层切片系统
复杂 Cutout 插件
```

原因：

- 当前核心是验证风格，不是建立复杂动画管线。
- 完整 Pose 图更适合 AI 生图和快速测试。
- Tween + Pose 切换已经可以做出纸片小剧场感。

### 14.3 翻面切 Pose 的实现逻辑

视觉逻辑：

```text
当前 Pose
→ scale.x 快速压缩到 0.05
→ 切换新 Pose 图
→ scale.x 展开到 1.0
```

可用于：

- 待机转攻击
- 攻击转待机
- 伙伴登场
- 技能释放
- 胜利动作切换

## 16. 背景美术风格

背景和角色不同，背景不需要纸片感，而是采用：

> **2D 动漫背景风格**

背景像动画小剧场中的舞台布景，为角色演出服务。

### 16.1 背景关键词

- 2D 动漫场景
- 清爽明亮
- 色彩柔和
- 轻奇幻
- 小剧场感
- 不写实
- 不 3D
- 不抢角色
- 信息量适中

### 16.2 背景与角色关系

画面关系应是：

> **背景像动画剧场的布景，角色像贴在舞台前方表演的纸片演员。**

要求：

- 背景可以有浅透视。
- 背景不要过度写实。
- 角色必须明显从背景中分离。
- 背景色彩不要压过角色。
- 战斗区域应保留干净空间。

### 16.3 动态背景元素

允许轻微背景动画：

- 云缓慢移动
- 旗帜轻微摆动
- 灯光轻闪
- 观众席剪影轻动
- 树影轻微摇晃

要求：

- 动态背景不能抢角色。
- 动画节奏要慢。
- 不要使用复杂 3D 运动。
- 不要制造视觉噪音。

## 18. 画面尺寸与角色尺寸规范

以 1920×1080 为基准。

### 18.1 战斗场景角色尺寸

| 元素 | 建议尺寸 |
|---|---|
| 主角站立高度 | 300-400px |
| 敌方主要角色高度 | 300-420px |
| 支援角色高度 | 主角的 80%-95% |
| Boss / 精英 | 主角的 120%-160% |
| 伤害数字 | 48-96px，根据重要程度变化 |
| 拟声词 | 80-180px，根据打击强度变化 |

### 18.2 安全区

重要信息应集中在画面中央安全区：

```text
左右各保留约 5%-8% 边距
上下各保留约 5%-10% 边距
底部 UI 区域不应遮挡角色脚底
```

## 19. 战斗站位规范

### 19.1 初版站位

建议初版战斗同屏可见：

```text
我方最多 3 人
敌方最多 3 人
```

### 19.2 站位原则

```text
我方：左侧 / 左下区域
敌方：右侧 / 右下区域
支援角色：从画面边缘滑入，不长期占主站位
```

### 19.3 多角色处理

为避免画面混乱：

- 当前行动角色亮度正常。
- 非当前行动角色可略微降低亮度或饱和。
- 后排角色可缩小到 85%-90%。
- 后排角色略微上移。
- 角色之间至少保留半个角色宽度的间距。
- 支援角色动作结束后可以退场。

## 20. 战斗表现方向

战斗核心观感：

> **Q版纸片角色在小剧场舞台上自动乱斗。**

### 20.1 战斗节奏

战斗应当：

- 短
- 快
- 有反馈
- 有看戏感
- 不需要玩家手动操作过多
- 自动战斗也要有观赏性

### 20.2 每回合演出结构

每回合可包含：

1. 待机小动作
2. 行动角色突出
3. Pose 翻面切换
4. 攻击位移 / 倾斜
5. 命中特效
6. 拟声词飞出
7. 伤害数字弹出
8. 战斗日志吐槽
9. 回到待机状态

## 21. 战斗反馈规范

### 21.1 命中反馈

推荐组合：

- 敌人后仰
- 敌人闪白
- 伤害数字弹出
- 拟声词飞出
- 速度线
- 镜头轻震
- 命中停顿
- 音效

### 21.2 伤害数字

建议：

| 类型 | 表现 |
|---|---|
| 普通伤害 | 清晰粗体，短促上浮 |
| 暴击 | 更大、更亮、带弹性放大 |
| 治疗 | 柔和上浮，可用绿色或暖色 |
| 护盾 | 带边框或盾牌小图标 |
| Miss | 漫画式偏移飞出 |

### 21.3 拟声词

拟声词是本项目的重要风格元素。

要求：

- 粗体
- 描边
- 可倾斜
- 快速出现
- 有弹性缩放
- 0.3-0.8 秒内消失
- 不长期遮挡角色

示例：

```text
啪！
砰！
斩！
咚！
嗖！
Duang!
哐！
```

## 29. 避免幼儿化的具体规则

由于角色是 2.5-3 头身，容易偏低龄化，因此需要避免：

- 低于 2 头身
- 眼睛过大到占脸部一半以上
- 四肢完全圆柱化
- 纯幼儿玩具感配色
- 没有职业特征的圆滚滚造型
- 过度奶嘴、婴儿化装饰

建议：

- 保留职业装备细节
- 保留清晰剪影
- 使用冒险者元素
- 表情可爱但不幼稚
- 姿势要有行动感

---

## 30. 避免擦边化的具体规则

本项目角色应保持健康、轻快、可爱、热血。

不建议：

- 暴露性服装
- 强调身体曲线
- 乳沟、胸部阴影线
- 内衣外穿
- 暧昧表情
- 过度脸红
- 仰视角擦边构图

允许：

- 可爱服装
- 职业制服
- 冒险者装备
- 健康活泼表情
- 少量害羞表情，但不能暧昧化

## 35. 最终风格结论

本项目不是传统逐帧动画，也不是复杂骨骼纸偶动画。

本项目的角色动画核心是：

> **60fps 播放 + 完整 Pose 切换 + 纸片翻面 + 整体模型倾斜 + 拟声词反馈 + 小剧场 UI 演出。**

最终目标是：

> **让玩家感觉自己在观看一场可互动的动漫纸片小剧场，而不是一个严肃写实的战斗模拟。**
