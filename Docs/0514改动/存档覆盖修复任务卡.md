# 存档覆盖任务卡：返回主菜单后存档未更新

## 问题描述
用户流程：
1. 已有存档1（旧游戏进度）
2. 开新游戏 → 推进到某层
3. 暂停菜单 → 返回主菜单（应触发主动存档，生成存档2）
4. 点击"继续游戏" → 加载的仍然是存档1（旧进度）

**预期行为：** 返回主菜单后，"继续游戏"应加载最新进度（存档2）。

## 根因排查方向

### 排查1：SaveManager 写文件是否覆盖旧存档

文件：`autoload/save_manager.gd`

确认 `save_run_state()` 使用 `FileAccess.WRITE` 模式（覆盖），不是 `FileAccess.READ_WRITE`：
```gdscript
func save_run_state(data: Dictionary, slot_id: int = 1) -> bool:
    var file_path: String = ConfigManager.SAVE_DIR + "save_%03d.json" % slot_id
    var file = FileAccess.open(file_path, FileAccess.WRITE)  # ← 必须是 WRITE
    if file == null:
        push_error("[SaveManager] 无法打开文件: %s" % file_path)
        return false
    var json_str: String = JSON.stringify(data)
    file.store_string(json_str)
    file.close()
    print("[SaveManager] 存档已保存: %s" % file_path)
    return true
```

如果是 `FileAccess.READ_WRITE`，会导致数据追加而不是覆盖。

### 排查2：返回主菜单时是否真正执行了存档

文件：`scenes/run_main/run_main.gd`

确认 `_on_return_main_menu()` 正确调用了 `_save_at_floor_entrance()`：
```gdscript
func _on_return_main_menu() -> void:
    print("[RunMain] 返回主菜单")
    if _run_controller != null:
        # 关键：必须先存档，再切场景！
        _run_controller._save_at_floor_entrance()
    get_tree().change_scene_to_file("res://scenes/main_menu/menu.tscn")
```

如果 `_run_controller` 为 null，或者 `_save_at_floor_entrance()` 内部有异常，存档就不会执行。

### 排查3：_save_at_floor_entrance 是否正确生成存档数据

文件：`scripts/systems/run_controller.gd`

确认 `_save_at_floor_entrance()` 调用 `SaveManager.save_run_state()`：
```gdscript
func _save_at_floor_entrance() -> void:
    var summary = get_current_run_summary()
    # 确保包含 node_options
    summary["node_options"] = _current_node_options
    SaveManager.save_run_state(summary)
    print("[RunController] 层入口存档: 第%d层" % _current_floor)
```

如果 `summary` 为空或 `_current_floor` 还是旧值，存档数据就是旧的。

### 排查4：GameManager.pending_save_data 是否被旧数据污染

文件：`autoload/game_manager.gd`

确认 `pending_save_data` 在以下时机被清空：
```gdscript
func _on_continue_game_requested() -> void:
    var save_data = SaveManager.load_latest_run()
    if not SaveManager.is_valid_save(save_data):
        return
    pending_save_data = save_data.duplicate()
    # ...
    change_scene("RUN_MAIN", "fade")
```

以及 `run_main.gd` 的 `_ready()` 中使用后清空：
```gdscript
func _ready() -> void:
    var pending_save = GameManager.pending_save_data
    if not pending_save.is_empty():
        # ... 使用存档恢复 ...
        GameManager.pending_save_data = {}  # ← 必须清空！
```

### 排查5：存档文件内容是否真正更新

在返回主菜单后，手动检查 `user://save_001.json` 的内容：
- `current_floor` 是否是当前层数？
- `hero.current_hp` 是否是当前HP？

如果文件内容确实是新的，但游戏加载的是旧的 → 问题在 `GameManager.pending_save_data` 缓存。
如果文件内容还是旧的 → 问题在 `SaveManager.save_run_state()` 没有覆盖。

## 修复步骤

### Step 1：验证 SaveManager 写模式

```gdscript
# save_manager.gd
func save_run_state(data: Dictionary, slot_id: int = 1) -> bool:
    var file_path: String = ConfigManager.SAVE_DIR + "save_%03d.json" % slot_id
    # 关键：WRITE 模式会覆盖旧文件
    var file = FileAccess.open(file_path, FileAccess.WRITE)
    if file == null:
        push_error("[SaveManager] Failed to open file: %s" % file_path)
        return false
    var json_str: String = JSON.stringify(data)
    file.store_string(json_str)
    file.close()
    print("[SaveManager] Saved to %s" % file_path)
    print("[SaveManager] Content: %s" % json_str.substr(0, 200))  # debug
    return true
```

### Step 2：验证返回主菜单时存档触发

在 `run_main.gd` `_on_return_main_menu()` 中加 debug：
```gdscript
func _on_return_main_menu() -> void:

```

### Step 3：验证继续游戏时加载最新存档

### Step 4：确保存档文件路径正确

确认 `ConfigManager.SAVE_DIR` 的值：
```gdscript
# 应该是 "user://saves/" 或类似路径
# 如果为空字符串，文件会写入错误位置
```

## 验证方法

1. 新开局 → 推进到第3层
2. 暂停菜单 → 返回主菜单
3. 查看 Godot Output 面板，确认有 `[RunMain] 存档完成` 输出
4. 检查 `user://save_001.json` 的 `current_floor` 是否为 3
5. 点击"继续游戏" → 确认回到第3层
6. 如果回到第1层（旧存档），说明存档未覆盖或加载逻辑有误
