## 核心改造点（基于真实高分代码）

### 1. StyleBoxFlat 阴影系统（向 MainMenu-MP 看齐）

高分项目的关键差异：**大量使用 shadow_size + shadow_offset 制造层次感**，而非单纯依赖边框。

```gdscript
# === 主菜单按钮样式（明亮版，参考 White.tscn） ===
func _create_btn_normal_style() -> StyleBoxFlat:
    var s := StyleBoxFlat.new()
    s.bg_color = Color(1, 1, 1, 1)                    # 纯白底
    s.border_color = Color(0.85, 0.85, 0.85, 1)       # 浅灰边框
    s.border_width_left = 2
    s.border_width_top = 2
    s.border_width_right = 2
    s.border_width_bottom = 2
    s.corner_radius_top_left = 8                        # 统一圆角
    s.corner_radius_top_right = 8
    s.corner_radius_bottom_left = 8
    s.corner_radius_bottom_right = 8
    s.shadow_color = Color(0, 0, 0, 0.08)             # 轻阴影
    s.shadow_size = 6
    s.shadow_offset = Vector2(0, 2)
    return s

func _create_btn_hover_style() -> StyleBoxFlat:
    var s := StyleBoxFlat.new()
    s.bg_color = Color(0.98, 0.98, 1, 1)              # 微蓝白
    s.border_color = Color(0.4, 0.6, 1, 1)            # 蓝色边框（MainMenu-MP 标准hover色）
    s.border_width_left = 2
    s.border_width_top = 2
    s.border_width_right = 2
    s.border_width_bottom = 2
    s.corner_radius_top_left = 8
    s.corner_radius_top_right = 8
    s.corner_radius_bottom_left = 8
    s.corner_radius_bottom_right = 8
    s.shadow_color = Color(0.4, 0.6, 1, 0.2)          # 蓝色投影
    s.shadow_size = 8                                   # 阴影扩散
    s.shadow_offset = Vector2(0, 3)
    return s

func _create_btn_pressed_style() -> StyleBoxFlat:
    var s := StyleBoxFlat.new()
    s.bg_color = Color(0.92, 0.94, 1, 1)
    s.border_color = Color(0.3, 0.5, 0.9, 1)
    s.border_width_left = 2
    s.border_width_top = 2
    s.border_width_right = 2
    s.border_width_bottom = 2
    s.corner_radius_top_left = 8
    s.corner_radius_top_right = 8

    s.corner_radius_bottom_right = 8
```

### 4. 背景系统

参考 MainMenu-MP 双层背景：
- **底层**：`ColorRect` 纯色底（明亮风格用 `Color(0.96, 0.97, 0.99, 1)` 极浅蓝白）
- **纹理层**：`TextureRect` 放背景图，`modulate = Color(0.88, 0.92, 0.98, 0.4)` 轻微淡蓝调 + 低透明度，不抢UI焦点

### 5. 入场动画（参考 Maaack 的 AnimationTree 方案）

Maaack 使用 `AnimationPlayer` + `AnimationTree` 做状态机动画：
- Intro → OpenMainMenu → OpenSubMenu
- 动画可跳过（任意输入跳过intro）

**简化版实现（用 Tween，不依赖AnimationTree）：**

```gdscript
func _play_entrance_animation() -> void:
    ## 初始态：所有UI元素下移+透明
    var elements: Array[Control] = [_title_label, _menu_buttons_container, _info_panel]
    for el in elements:
        el.modulate.a = 0.0
        el.position.y += 30
    
    ## 顺序入场
    var tween := create_tween().set_trans(Tween.TRANS_QUAD).set_ease(Tween.EASE_OUT)
    for i in range(elements.size()):
        var delay := i * 0.12
        tween.tween_property(elements[i], "modulate:a", 1.0, 0.4).set_delay(delay)
        tween.parallel().tween_property(elements[i], "position:y", elements[i].position.y - 30, 0.45).set_delay(delay)
```

### 6. 按钮点击反馈（弹性动画）

```gdscript
func _on_button_pressed(button: Button) -> void:
    var tween := create_tween().set_trans(Tween.TRANS_BACK).set_ease(Tween.EASE_OUT)
    tween.tween_property(button, "scale", Vector2(0.95, 0.95), 0.05)
    tween.tween_property(button, "scale", Vector2.ONE, 0.15)
```

### 7. 子菜单系统（参考 Maaack 封装）

```gdscript
var _sub_menu: Control = null

func _open_sub_menu(scene: PackedScene) -> void:
    _sub_menu = scene.instantiate()
    add_child(_sub_menu)
    _main_container.hide()
    _sub_menu.tree_exiting.connect(_close_sub_menu, CONNECT_ONE_SHOT)

func _close_sub_menu() -> void:
    if _sub_menu == null:
        return
    _sub_menu.queue_free()
    _sub_menu = null
    _main_container.show()
```

## 实际改造清单

### menu.tscn 节点结构调整

```
MainMenu (Control)
├── BackgroundLayer (CanvasLayer, layer=0)
│   ├── ColorRect          # 纯色底，Color(0.96, 0.97, 0.99, 1)
│   └── BackgroundTexture  # TextureRect，背景图，modulate淡蓝白低透明度
├── UILayer (CanvasLayer, layer=2)
│   ├── MainContainer (Control, anchors_preset=15)
│   │   ├── TitleLabel     # 游戏标题
│   │   ├── MenuButtons    # VBoxContainer，主按钮组
│   │   ├── IconBar        # HBoxContainer，功能图标（档案/PVP/商店/排行）
│   │   └── InfoPanels     # 信息面板区（底部左右）
│   └── VersionLabel       # 右下角版本号
└── TransitionOverlay      # ColorRect，全屏遮罩，用于场景切换
```

### menu.gd 核心修改

1. **删除**：所有暗黑霓虹配色常量（`COLOR_BTN_BG`、`COLOR_BTN_BORDER` 等）
2. **新增**：`Theme` 资源统一管理按钮样式，或用 `_create_*_style()` 函数动态创建
3. **修改**：`_setup_button_style()` 改用上述 StyleBoxFlat 阴影系统
4. **新增**：`_play_entrance_animation()` 入场动画
5. **新增**：`_on_button_pressed()` 弹性点击反馈
6. **修改**：背景色调为极浅蓝白
7. **保留**：TextureButton 图片按钮的 hover 切换逻辑（如果有图片资源）

### 氛围粒子（樱花/尘埃）保留

已有实现很好，只需微调颜色适配明亮背景：
- 樱花粉：`Color(1.0, 0.72, 0.77, 0.6)` → 降低透明度到 `0.4`，避免在白色背景上过艳
- 尘埃：`Color(1.0, 0.95, 0.85, 0.25)` → 保持不变，暖白在明亮背景上更自然

## 测试验收标准

- [ ] 背景：极浅蓝白纯色 + 背景图淡蓝调低透明度，不压抑
- [ ] 按钮：纯白底 + 浅灰边框 + 8px圆角 + 轻阴影
- [ ] Hover：边框变蓝 + 阴影扩散 + 字体变蓝
- [ ] 按下：按钮缩小到0.95 → 弹性回弹
- [ ] 面板：白底 + 大圆角(12px) + 重阴影(15px)
- [ ] 标题：56px + 近黑 + 轻微outline/shadow
- [ ] 入场：标题→按钮→面板依次淡入+上滑，间隔0.12s
- [ ] 粒子：樱花降低透明度，暖白尘埃自然
- [ ] 子菜单：打开时主菜单隐藏，关闭时恢复
- [ ] 1280x720 和 1920x1080 无重叠
