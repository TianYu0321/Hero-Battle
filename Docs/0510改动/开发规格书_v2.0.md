# 《赛马娘版 Q宠大乐斗》开发规格书

> 文档版本：v2.0 | 基于设计文档初稿 v0.9 + 0509修正 | 编制日期：2026-05-09
> 目标引擎：Godot 4.x | 平台：PC（Windows/Mac/Linux） | 风格：2D 像素风

## 一、项目概述

### 1.1 游戏定位

融合型 Roguelike 策略养成游戏，核心体验为**"局内养成规划 + 局外 PVP 验证"的双循环结构**。单局内通过 30 层的爬塔式养成构筑角色，通过局内 PVP 检定和终局战验证构筑强度，最终生成"终局斗士档案"参与局外异步 PVP。

### 1.2 核心体验

| 维度 | 描述 |
|:---|:---|
| **单局体验** | 选择主角与初始伙伴 → 30 层爬塔（训练/战斗/休息/外出事件/救援/PVP） → 终局战 → 生成斗士档案 |
| **单局时长** | 25-35 分钟 |
| **核心爽感** | 前期规划 → 中期流派成型 → 终局自动战斗中看到自己构筑的实际表现（连锁爆发） |
| **重刷动力** | 多主角 × 多伙伴 × 多流派 × 终局斗士强度排名 |
| **局外驱动力** | 异步 PVP 排名、伙伴/主角解锁、斗士档案优化 |

### 1.3 已确认的设计决策

| # | 决策项 | 确认方案 | 决策理由 |
|:---:|:---|:---|:---|
| 1 | **MVP 范围** | **坚持原版**：3 主角 + 12 伙伴 + 30 回合 + 7 类节点 + 终局保存 + 局外 PVP + 解锁系统 | 完整验证双循环体验 |
| 2 | **普通战斗结算** | **放弃预结算**，所有战斗统一走完整自动战斗逻辑，通过**播放模式**区分体验速度 | 降低维护成本 + 保留养成反馈 |
| 3 | **PVP 服务端方案** | **混合渐进**：Phase 1 不做 PVP → Phase 2 本地 AI 对手池 → Phase 3 BaaS（Firebase） | 无后端经验，渐进降低风险 |
| 4 | **目标平台** | **PC 优先**（Windows/Mac/Linux），暂不包含移动端 | 聚焦核心验证，分辨率更宽松 |
| 5 | **属性熟练度初版** | **保守设计**：专精阶段加成从 +9 降至 +5，测试后再决定是否加强 | 防止滚雪球失控 |
| 6 | **伙伴 Lv5 机制** | **Lv3 和 Lv5 都是质变**，主角也有 Lv3/Lv5 质变 | 技能与训练额外效果 |

### 1.4 关键成功指标

| 阶段 | 验证指标 |
|:---|:---|
| Phase 1 结束 | 可完整跑通 30 回合单局，终局战斗有爽感 |
| Phase 2 结束 | 3 主角 × 多流派胜率在 40%-60% 之间 |
| Phase 3 结束 | D7 留存 ≥ 30% |
| Phase 4 结束 | 外部测试评分 ≥ 7/10 |

## 二、技术架构

### 2.1 引擎与工具

| 项目 | 选型 | 说明 |
|:---|:---|:---|
| 游戏引擎 | Godot 4.x（.NET/C# 或 GDScript） | 2D 像素风首选，内置 Resource 系统适合数据驱动 |

#### 每层选项结构（核心界面）

**重要：每层选项为两层选择结构，不是单层随机事件。**

**第一层 - 主选项（每层固定4个按钮）**：

```
┌─────────────────────────────────────────────────────────┐
│  第 X 层                                                 │
│                                                         │
│  [ 训练 ]  [ 战斗 ]  [ 休息 ]  [ 外出 ]                │
│                                                         │
│  点击后展开第二层（仅训练按钮）                          │
└─────────────────────────────────────────────────────────┘
```

| 主选项 | 说明 |
|:---|:---|
| **训练** | 点击后**展开**五属性训练面板（见第二层），选择具体属性进行训练 |
| **战斗** | 直接进入普通战斗 |
| **休息** | 直接执行，恢复主角最大生命×15%，本层结束 |
| **外出** | 触发随机事件（休息恢复/商店/陷阱/回复/特殊事件/精英战），事件类型按权重随机 |

**第二层 - 训练面板（点击"训练"后展开，纵向排列）**：

```
┌─────────────────────────────────────────┐
│  选择训练属性         [返回]             │
│                                         │
│  [ 体魄 ]  LV:3    [盾卫头像][牧师头像] │
│  [ 力量 ]  LV:2    [剑士头像]            │
│  [ 敏捷 ]  LV:1                          │
│  [ 技巧 ]  LV:4    [猎人头像][术士头像] │
│  [ 精神 ]  LV:2    [药师头像]            │
│                                         │
│  点击属性行进行训练                      │
└─────────────────────────────────────────┘
```

**训练面板规则**：

| 规则 | 说明 |
|:---|:---|
| 纵向排列 | 5种属性训练**纵向排列**，每行独立显示 |
| 等级显示 | 每行显示当前训练等级 **LV:x** |
| 伙伴出现 | 已入队伙伴**随机**出现在训练项上（每属性0-2名伙伴） |
| 擅长属性概率 | 伙伴出现在自己**擅长属性**上的概率 = 60%，出现在其他属性上 = 各10% |
| 同行伙伴加成 | 2名同行伙伴必定出现在训练面板上（擅长属性优先） |
| 支援效果 | 点击有伙伴的训练项时，该伙伴提供**额外属性加成**（固定值，不影响副属性） |
| 训练等级加成 | 该属性训练次数越多，训练等级越高，本次训练收益越高（见训练等级系统） |

**休息规则**：

| 项目 | 规格 |
|:---|:---|
| 效果 | 恢复主角最大生命 × 15% |
| 特殊 | 若有"休息触发"型伙伴，额外执行其效果 |
| 限制 | **无次数限制** |

**外出事件池**：

| 事件类型 | 权重 | 说明 |
|:---|:---:|:---|
| 休息恢复 | 30% | 恢复最大生命×15% |
| 商店 | 20% | 打开商店界面，升级主角或伙伴 |
| 陷阱 | 15% | 损失生命或属性，无收益 |
| 回复泉 | 15% | 发现回复泉，恢复生命 |
| 特殊事件 | 15% | 随机正面/中性事件 |
| 精英战 | 5% | 高风险战斗，可选放弃 |

**关键区分**：
- **训练** = 主动养成，玩家选择属性，伙伴支援加成
- **战斗** = 直接进入普通战斗
- **休息** = 纯回复，无养成收益
- **外出** = 随机事件，可能休息/商店/陷阱，有随机性

### 4.3 统一自动战斗引擎规格

**核心原则：所有战斗类型共用一套逻辑，通过播放模式区分体验。**

#### 战斗流程（20 回合）

```
每回合流程：
1. 行动顺序判定（基于"速度"属性 + 随机波动，速度受招式/敏捷等效果影响）
2. 主角普攻/技能
3. 检查伙伴援助触发条件（遍历全部 5 名援助伙伴）
4. 执行触发的援助效果
5. 连锁触发检查（不限制段数，满足条件即可触发）
6. 状态结算（BUFF/DEBUFF 持续回合递减）
7. 必杀技触发检查（每场限 1 次）
8. 回合结束判定（回合内所有行动完毕后，再轮到对方回合）
```

#### 播放模式

**所有战斗类型统一标准播放，不限时间。站位为传统RPG：左侧友方，右侧敌方。**

| 战斗类型 | 播放模式 | 展示内容 |
|:---|:---|:---|
| **普通战斗** | 标准播放 | 双方立绘 + 血条 + 每回合伤害数字 + 最终胜负 |
| **精英战** | 标准播放 | 完整动画 + 伙伴援助 + 必杀技 + 连锁展示 |
| **PVP 检定** | 标准播放 | 完整动画 + 伙伴援助 + 连锁展示 + 胜负判定 |
| **终局战** | 标准播放 | 完整动画 + 连锁展示 + 胜负判定 |

#### 伤害公式（配置驱动）

```
伤害 = 基础值 × 属性系数 × 技能倍率 × 随机波动(0.9-1.1)

属性系数计算：
  - 攻击方：力量 × 力量系数 + 技巧 × 技巧系数
  - 防御方：体魄 × 体魄系数
  - 敏捷影响：先手概率、闪避概率、行动频率
  - 精神影响：抗性、低血时伤害加成
```

所有系数通过 BattleFormulaConfig 配置表驱动，便于平衡调整。

### 4.7 主角技能设计（已定稿）

#### 三主角统一设计原则

| 原则 | 说明 |
|:---|:---|
| **初始值差异** | 三主角仅在初始五维属性上有差异，各自突出一个主属性 |
| **无成长差异** | 属性成长完全由**锻炼次数**和**伙伴支援**决定，三主角成长空间完全相同 |
| **机制差异** | 三主角的核心差异在于**技能触发时机**和**攻击段数设计** |
| **段数体系** | 勇者（1段大数字）→ 影舞者（2-4段连击）→ 铁卫（受击反击1段），**段数是成型build的核心维度** |

**三主角核心差异一览**：

| 维度 | 勇者 | 影舞者 | 铁卫 |
|:---|:---|:---|:---|
| **最高初始属性** | 力量 16 | 敏捷 16 | 体魄 16 |
| **攻击段数** | 1段（一击） | 2-4段（连击） | 0-1段（受击反击） |
| **技能触发时机** | 攻击后 | 每击自动 | 受击后 |
| **必杀触发** | 敌方40%血 | 第8回合 | 自己50%血 |
| **build核心** | 力量→大数字 | 敏捷→多段→触发伙伴 | 体魄→站住→伙伴输出 |

**核心属性**：力量（主）+ 技巧（副）

> **重要设计原则**：主角之间**仅初始值不同**，属性成长完全由**锻炼次数**和**伙伴支援**决定。勇者的力量初始最高，意味着开局就有最高的基础攻击力，但后续成长空间与其他主角相同。

|:---|:---|
| 触发类型 | 条件触发（敌方生命值首次低于40%时） |
| 触发时机 | 主角行动回合内，满足条件立即触发 |
| 效果 | 造成 **300% 攻击力** 的巨额伤害 |
| 伤害公式 | 力量 × 3.0 × 随机波动(0.9-1.1) |
| 特殊规则 | 无视敌方30%防御力（确保斩杀感） |
| 整场限制 | **限 1 次** |
| 视觉表现 | 勇者跃起 → 剑聚光 → 巨大斩击 → 大数字 + 屏幕震动 |
| 设计意图 | 斩杀爽感，"敌人快死了，一剑带走" |

**技能联动闭环**

```
力量越高 → 普攻伤害越高 → 追击斩伤害越高
                                ↓
                        更快将敌方压至40%以下
                                ↓
                        触发终结一击（300%伤害秒杀）
                                ↓
                        大数字爽感 + 战斗快速结束
```

**与伙伴的联动方向**：
- 配合"暴击后触发"型伙伴：追击斩提供更多攻击次数 → 更多暴击机会 → 更多连锁
- 配合"低血增伤"型伙伴：终结一击将敌方压至低血 → 触发伙伴的斩杀型援助

#### 铁卫 - 体魄型防守反击

**角色定位**：与勇者和影舞者形成三角对比——勇者"主动一击"、影舞者"主动多段"、铁卫"被动防守"。核心体验是"你打不动我，我耗死你"。

**核心属性**：体魄（主）+ 精神（副）

**设计理念**：
- 勇者和影舞者本质都是**主动进攻**，靠自己输出打伤害
- 铁卫是**被动防守**，靠自己站住，让伙伴来打伤害
- **PVP优势**：高坦度+反击+控制，极难被秒杀，拖越久越强

> **重要**：主角间仅初始值不同，成长全靠锻炼和伙伴。铁卫生存优势来自开局16点体魄，但后续体魄成长与其他主角相同。

**初始属性**：

| 属性 | 初始值 | 说明 |
|:---|:---:|:---|
| **体魄** | **16** | **核心属性，三主角中最高初始值** |
| 力量 | 8 | 偏低，非核心输出 |
| 敏捷 | 10 | 偏低，后手居多 |
| 技巧 | 10 | 标准 |
| **精神** | **14** | **副属性，影响反击概率和抗性** |

---

**铁卫构筑方向**

铁卫的局内构筑有两个清晰的前中期方向：

| 阶段 | 策略 | 说明 |
|:---|:---|:---|
| **前期** | 寻找**高战力伙伴**度过精英事件 | 铁卫自身输出低（力量初始8），必须靠伙伴打伤害 |
| **前期** | 寻找**高锻炼支援伙伴**提升属性数值 | 通过锻炼堆高体魄和精神，确保坦度足够触发反击 |
| **商店决策** | 升级自己（更肉）or 升级伙伴（更强输出） | 资源分配的核心痛苦抉择 |

**关键设计洞察**：
- 伙伴的质变在 Lv3 和 Lv5，低级伙伴的**数值成长**往往比高级技能更实用
- 1 个 Lv3 的铁卫已经有足够的坦度来频繁触发反击，不一定需要追求 Lv5
- 铁卫的构筑核心是"**我能挨多少打，我的伙伴就能输出多少**"——坦度=输出频率
- 防御反击**算作攻击段数**，可以通过调整段数来平衡强度（目前设计为受击后1段反击）

---

**常规技能 - 铁壁反击**

| 项目 | 规格 |
|:---|:---|
| 触发类型 | 概率触发（主角每次受击后） |
| 基础触发概率 | 25% |
| 精神加成 | 精神每+10点，概率+2%，上限40% |
| 效果 | **反弹**受到伤害的 **50%** 给攻击者 |
| 特殊效果 | 反击有 **10% 概率** 使敌方 **眩晕 1 回合**（眩晕期间跳过行动） |
| 伤害公式 | 敌方本次造成伤害 × 0.5 |
| 连锁标签 | 反击（**算作1段攻击**，可触发连锁触发型的伙伴援助） |
| 每场上限 | 无 |
| 视觉表现 | 受击时盾牌闪烁金光，一道反击光芒射向敌方，敌方头顶出现短暂眩晕星号 |
| 设计意图 | **防守即攻击**——越挨打越反击。反击算作**1段攻击**，意味着铁卫每受击1次=产生1段攻击判定，与影舞者的多段攻击形成呼应 |
| 强度调节 | 可通过调整"反击是否算段数"来平衡：算段数=偏强（触发伙伴更多），不算段数=偏弱 |

**必杀技 - 不动如山**

| 项目 | 规格 |
|:---|:---|

| 效果 | 开启持续 **3 回合** 的防御姿态 |
| 减伤 | 期间受到的伤害 **-40%** |
| 反击强化 | 期间每次受击 **必定 100% 触发** 铁壁反击（原25%→100%） |
| 击晕强化 | 期间反击击晕概率从 10% 提升至 **25%** |
| 整场限制 | **限 1 次** |
| 视觉表现 | 全身笼罩金色护盾，脚下出现阵法特效，持续3回合，每次反击时盾牌爆发出强光 |
| 设计意图 | "绝境反击"——半血以下进入绝对防御状态，挨打必反击，概率击晕控场 |

**不动如山的联动爽感**：

```
敌方攻击铁卫（3回合buff期间）：
  → 伤害-40%（铁卫只受60%伤害）
  → 必定触发铁壁反击（100%反弹50%伤害）
  → 25%概率击晕敌方（敌方跳过下回合）

一场典型的不动如山体验：
  敌方回合1：攻击 → 被减伤 → 被反击反弹 → 运气不好被击晕
  敌方回合2：跳过（眩晕）
  敌方回合3：攻击 → 被减伤 → 被反击反弹
  
3回合内敌方实际只出手了2次，且每次都被反伤
→ 铁卫几乎无伤，敌方被自己打残
→ "你打不动我"的爽感
```

**核心属性**：敏捷（主）+ 力量（副）

**设计理念**：
- 勇者的追击斩 = 30%概率触发一次额外大伤害 → **"我这一刀很痛"**
- 影舞者的连击 = 每次普攻分裂多段 → 每段都有一次触发伙伴的机会 → **"我一直在砍，伙伴一直在动"**

> **重要**：主角间仅初始值不同，成长全靠锻炼和伙伴。

**初始属性**：

**常规技能 - 疾风连击**

| 项目 | 规格 |
|:---|:---|
| 触发类型 | 被动常驻（每次普通攻击自动生效） |
| 效果 | 每次普攻分裂为 **2-4 段** 小伤害（每段 = 攻击力 × 0.35） |
| 段数计算 | 基础 2 段 + 敏捷每+15点多1段，上限 4 段 |
| 总伤害对比 | 4段 × 35% = 140% 攻击力（略高于勇者普攻100%） |
| 关键设计 | **每段伤害独立判定**伙伴援助触发条件 |
| 视觉表现 | 主角快速连续出剑，每段一个较小的伤害数字飘出 |
| 设计意图 | **多段 = 多触发机会**——伙伴看到4次攻击，可能触发4次援助 |

|:---|:---|:---|

| 触发"暴击后"型伙伴 | 暴击=1次判定 | 暴击可能多次，多次判定 |
| 连锁体验 | 爆发型（一击一个大数字） | 连发型（数字一直在跳） |
| 视觉感受 | 一剑重击 | 剑影缭乱 |

```
  → 6次攻击 × 45% × 2名伙伴 = 平均 5.4 次伙伴援助触发

一场风暴乱舞中，伙伴可能连续触发 5+ 次援助 → 满屏伤害数字
→ "伙伴跟我一起战斗"的团队感达到顶峰
```

#### 精英战奖励分档（3选1）

**奖励原则**：难度越高，奖励池越丰厚，稀有选项出现概率越高。

| 难度层 | 金币奖励 | 强力BUFF选项 | 大量金币选项 | 稀有道具选项 |
|:---:|:---:|:---|:---|:---|
| **1（重甲守卫）** | 30-50 | 攻击+10%（3回合） | 金币+80 | 生命回复+20 |
| **2（暗影刺客）** | 50-80 | 攻击+15%（4回合） | 金币+120 | 生命回复+30 |
| **3（元素法师）** | 80-120 | 攻击+20%（5回合） | 金币+180 | 下回合必定先手 |
| **4（狂战士）** | 120-180 | 攻击+25%（5回合）+ 暴击+10% | 金币+250 | 终结一击触发阈值提升至50% |
| **5（混沌领主）** | 180-250 | 攻击+30%（6回合）+ 全属性+5% | 金币+350 | 本场战斗伙伴援助触发概率+15% |

**奖励池生成规则**：
- 3 个选项从 3 个类别中各随机抽取 1 个（强力BUFF / 大量金币 / 稀有道具）
- 奖励数值在范围内根据主角当前属性浮动（属性越高浮动上限越高）
- 难度4-5的稀有道具为**战斗机制类**（改变规则），区别于低难度的数值类道具

### 5.3 PVP 检定

| 项目 | 规格 |
|:---|:---|
| 触发 | 第 10/20 层固定触发 |
| 对手匹配 | 异步匹配同层玩家影子；无匹配则使用AI对手 |
| AI规格 | 固定角色（固定五维数值）+ 随机伙伴（均为等级1） |
| 战斗模式 | 标准播放 |
| 失败惩罚 | **仅影响奖励**，不淘汰本局，无后续惩罚 |
| 累积惩罚 | **无** |

#### PVP 胜利奖励（偏向直接强化和滚动优势）

| 奖励类型 | 说明 |
|:---|:---|
| **主角升级** | 主角等级+1 |
| **高额属性成长** | 全属性+3~5（或指定主属性+8~10） |
| **生命回复/上限** | 恢复全部生命 或 最大生命+10% |
| **高额局内金币** | +80~150金币 |
| **伙伴升级** | 随机1名伙伴等级+1 |
| **训练强化** | 随机1种属性训练等级+1 |

**第10层PVP胜利**：从上述奖励池中随机抽取3项，玩家三选一
**第20层PVP胜利**：从上述奖励池中随机抽取3项（数值更高），玩家三选一

#### PVP 失败奖励（基础奖励 + 追赶补偿）

| 奖励类型 | 说明 |
|:---|:---|
| **少量属性成长** | 全属性+1~2 |
| **少量局内金币** | +20~40金币 |
| **事件可视化** | 获得接下来3层的节点预览能力 |
| **追赶型BUFF** | 下3层训练收益+20% 或 下3层金币掉落+30% |

**事件可视化说明**：
- 失败后玩家可提前看到接下来3层的部分事件类型
- 帮助玩家更精确地选择训练/战斗/休息/外出路线
- PVP失败不是纯惩罚，而是提供弯道超车机会

**第10层PVP失败**：少量属性+少量金币+事件可视化（3层）
**第20层PVP失败**：少量属性+少量金币+事件可视化（3层）+ 追赶型BUFF

**终局PVP（第30层）**：影响通关评价和斗士档案评分 |

### 6.3 通关分数计算（已定稿）

#### 总分公式

```
总分 = 终局战表现分 × 0.4
     + 养成效率分 × 0.2
     + PVP 表现分 × 0.2
     + 流派纯度分 × 0.1
     + 连锁展示分 × 0.1
```

#### 各项分数详细计算

**1. 终局战表现分（权重40%）**

| 子项 | 计算方式 | 满分 |
|:---|:---|:---:|
| 终局战胜利 | 胜利=50分，失败=0分 | 50 |
| 剩余生命比例 | （终局战后剩余生命/最大生命）× 30 | 30 |
| 终局战造成伤害 | （实际伤害/敌方最大生命）× 20 | 20 |
| **小计** | | **100** |

**2. 养成效率分（权重20%）**

| 子项 | 计算方式 | 满分 |
|:---|:---|:---:|
| 总属性成长 | （终局五维总和 / 初始五维总和 - 1）× 30 | 30 |
| 金币使用效率 | （实际用于升级的金币 / 获得的总金币）× 30 | 30 |
| 属性均衡度 | 五维标准差的倒数 × 20（越均衡分越高） | 20 |
| 精英战胜率 | （精英战胜利次数 / 精英战总次数）× 20 | 20 |
| **小计** | | **100** |

**3. PVP表现分（权重20%）**

| 子项 | 计算方式 | 满分 |
|:---|:---|:---:|
| 第10回PVP | 胜利=40分，失败=15分，跳过=0分 | 40 |
| 第20回PVP | 胜利=40分，失败=15分，跳过=0分 | 40 |
| PVP造成伤害 | 两场PVP总伤害 / 参考值 × 20 | 20 |
| **小计** | | **100** |

**4. 流派纯度分（权重10%）**

| 子项 | 计算方式 | 满分 |
|:---|:---|:---:|:---:|
| 主属性占比 | 最高属性 / 五维总和 × 50 | 50 |
| 技能触发次数 | 常规技能+必杀技总触发次数 / 参考值 × 30 | 30 |
| 伙伴协同 | 伙伴援助总触发次数 / 参考值 × 20 | 20 |
| **小计** | | **100** |

**5. 连锁展示分（权重10%）**

| 子项 | 计算方式 | 满分 |
|:---|:---|:---:|
| 最大连锁段数 | 单场最高CHAIN数 × 10（上限4段=40分） | 40 |
| 总连锁次数 | 全战斗连锁总次数 / 参考值 × 30 | 30 |
| 伙伴援助总次数 | 全战斗伙伴援助触发总次数 / 参考值 × 30 | 30 |
| **小计** | | **100** |

#### 评级标准

| 评级 | 总分范围 | 说明 |
|:---:|:---:|:---|
| **S** | ≥ 90 | 完美通关，构筑和操作都达到极高水平 |
| **A** | 75-89 | 优秀通关，构筑合理，有少量优化空间 |
| **B** | 60-74 | 良好通关，完成基本目标 |
| **C** | 40-59 | 勉强通关，构筑有明显缺陷 |
| **D** | < 40 | 失败或未通关 |

#### 分数设计意图

- **终局战40%权重**：终局战是整局养成的最终验证，最重要的分数来源
- **PVP20%权重**：PVP是中期检定，表现好坏直接影响总评
- **养成效率20%权重**：鼓励合理规划资源，而非无脑堆单一属性
- **流派纯度10%**：鼓励专精一个build方向，而非平均分配
- **连锁展示10%**：鼓励触发华丽的伙伴连锁，提升观赏性

### 6.4 通关分数计算（已定稿）

```
总分 = 终局战表现分 × 0.4
     + 五维属性分 × 0.25
     + 伙伴/主角等级分 × 0.2
     + 剩余金币分 × 0.15
```

**各项详细计算**：

| 子项 | 权重 | 计算方式 | 满分 |
|:---|:---:|:---|:---:|
| **终局战表现分** | 40% | 终局战胜利=50，剩余生命比例×30，造成伤害比例×20 | 100 |
| **五维属性分** | 25% | 终局五维总和 / 参考值 × 100 | 100 |
| **伙伴/主角等级分** | 20% | （主角等级×5 + 所有伙伴等级之和×3）/ 参考值 × 100 | 100 |
| **剩余金币分** | 15% | 剩余金币 / 获得总金币 × 100 | 100 |

## 七、第4主角：术士（精神型状态控制）

### 7.1 术士设计（Phase 2扩展 - 已定稿）

**角色定位**：状态控制型，与前三者形成四角关系——勇者（一击）、影舞者（连击）、铁卫（防守）、术士（控场）。核心体验是"削弱敌人，让敌人自我毁灭"。

**核心属性**：精神（主）+ 技巧（副）

**设计理念**：
- 勇者是"我直接砍死你"，术士是"我让你自己毒死自己"
- 精神属性影响DEBUFF强度、持续回合和触发概率
- 术士的构筑核心是"堆精神→DEBUFF更强→敌人越弱→战斗越轻松"
- **独特机制**：术士的DEBUFF可以叠加，叠加后产生质变效果

**初始属性**（仅初始值差异）：

| 属性 | 初始值 | 说明 |
|:---|:---:|:---|
| 体魄 | 10 | 偏低，脆皮 |
| 力量 | 8 | 极低，几乎不输出 |
| 敏捷 | 10 | 标准 |
| 技巧 | 12 | 副属性，影响DEBUFF命中率 |
| **精神** | **16** | **核心属性，三主角中最高初始值** |

---

**常规技能 - 灵魂灼烧**

| 项目 | 规格 |
|:---|:---|
| 触发类型 | 被动常驻（每次主角普通攻击自动附加） |
| 效果 | 普攻附加**灵魂灼烧**DEBUFF：每回合结束时扣除敌方最大生命×3% |
| 叠加规则 | 每次普攻叠加1层，最多叠加**5层**，层数越高伤害越高 |
| 层数伤害 | 1层=3%, 2层=5%, 3层=8%, 4层=12%, 5层=18%（每层额外叠加递增） |
| 持续回合 | 3回合（可刷新） |
| 精神加成 | 精神每+10点，每层基础伤害+0.5% |
| 视觉表现 | 敌方身上叠加暗红色火焰层数标记，每层一个小火焰 |
| 设计意图 | **持续DOT伤害**——战斗越长敌人掉血越多，术士不怕拖 |

---

**必杀技 - 灵魂收割**

| 项目 | 规格 |
|:---|:---|
| 触发类型 | 条件触发（敌方身上有≥3层灵魂灼烧时） |
| 触发时机 | 主角行动回合内，满足条件立即触发 |
| 效果 | **引爆**所有灵魂灼烧层数，造成层数×精神×2.0的爆发伤害 |
| 特殊规则 | 引爆后层数清零，但敌方获得"灵魂虚弱"（受到的伤害+20%，2回合） |
| 整场限制 | **限 1 次** |
| 视觉表现 | 敌方身上所有火焰汇聚成一个巨大的骷髅头爆炸 → 大数字伤害 |
| 设计意图 | "让DEBUFF变现"——持续叠加后的终极爆发，层数越高收割越痛 |

**灵魂收割的伤害对比**：

| 层数 | 灼烧总伤害（3回合） | 收割伤害 | 合计 |
|:---:|:---:|:---:|:---:|
| 1层 | 9% | 精神×2.0 | 低 |
| 2层 | 15% | 精神×4.0 | 中 |
| 3层 | 24% | 精神×6.0 | 高 |
| 4层 | 36% | 精神×8.0 | 极高 |
| 5层 | 54% | 精神×10.0 | 秒杀级 |

**关键设计**：
- 3层是触发收割的最低门槛，但5层才是最大收益
- 玩家面临抉择：3层就收割（安全）vs 叠到5层再收割（高风险高收益）
- 敌方持续3回合不掉层，意味着要在3回合内叠到目标层数

---

**技能联动闭环**

```
精神越高 → 灼烧每层伤害越高
    → 叠加层数越快达到目标
    → 收割伤害越高
    → 灵魂虚弱（+20%受伤）→ 后续普攻更痛
    → 更快叠满5层
    → 形成"越打越痛"的正循环
```
