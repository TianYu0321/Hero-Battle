# 任务卡：角色精灵图路径硬编码 → ConfigManager 统一配置

## 背景

当前 `sprite_path` 的获取方式：
- `scenes/run_main/run_main.gd` → `_on_battle_ended()` 里用 `match hero_config_id` 硬编码写死路径
- 只有 `hero_config_id == 2`（shinobi）有配置，其他英雄为空字符串
- 敌人 sprite_path 当前写死或缺失
- 新增英雄需要改 `run_main.gd`，违反"数据驱动"原则

**目标**：所有角色/敌人的精灵图路径、动画配置、切割参数统一从 `ConfigManager` 读取，UI 层只负责 `load()` 和显示。

## 执行步骤

### Step 1：ConfigManager 新增 sprite 配置接口

**文件**：查找并修改 `scripts/autoload/config_manager.gd`（或类似位置）

**新增内容**：

```gdscript
## 英雄精灵图路径配置
const HERO_SPRITE_PATHS: Dictionary = {
    1: "res://assets/characters/warrior/hero_frames.tres",
    2: "res://assets/characters/shinobi/hero_frames.tres",
    3: "res://assets/characters/paladin/hero_frames.tres",
    # 后续英雄继续追加，key 是 hero_config_id
}

## 敌人精灵图路径配置
## 支持两种模式：
## 1. 按 enemy_config_id 直接映射
## 2. 按 enemy_type / 模板名映射
const ENEMY_SPRITE_PATHS: Dictionary = {
    2001: "res://assets/characters/enemies/slime_frames.tres",
    2002: "res://assets/characters/enemies/goblin_frames.tres",
    2003: "res://assets/characters/enemies/wolf_frames.tres",
    2004: "res://assets/characters/enemies/elite_knight_frames.tres",
    2005: "res://assets/characters/enemies/chaos_lord_frames.tres",
    # 后续敌人继续追加
}

## 默认精灵图路径（找不到配置时的 fallback）
const DEFAULT_HERO_SPRITE: String = "res://assets/characters/default_hero_frames.tres"
const DEFAULT_ENEMY_SPRITE: String = "res://assets/characters/enemies/default_enemy_frames.tres"

## 获取英雄精灵图路径
static func get_hero_sprite_path(hero_config_id: int) -> String:
    return HERO_SPRITE_PATHS.get(hero_config_id, DEFAULT_HERO_SPRITE)

## 获取敌人精灵图路径
static func get_enemy_sprite_path(enemy_config_id: int) -> String:
    return ENEMY_SPRITE_PATHS.get(enemy_config_id, DEFAULT_ENEMY_SPRITE)

## 获取精灵图动画帧配置
## 返回: {"hframes": int, "vframes": int, "animations": {"idle": [0,1,2], "attack_1": [3,4], ...}}
static func get_sprite_anim_config(path: String) -> Dictionary:
    ## 如果配置了动画切割，返回详细配置；否则返回默认值
    ## 当前默认: 4x4 切割，idle=0~3, attack_1=4~7, hurt=8~11, dead=12~15
    return {
        "hframes": 4,
        "vframes": 4,
        "animations": {
            "idle": [0, 1, 2, 3],
            "attack_1": [4, 5, 6, 7],
            "hurt": [8, 9, 10, 11],
            "dead": [12, 13, 14, 15],
        }
    }
```

**注意**：
- `ConfigManager` 当前是否已经是 `@tool` / `class_name` / `extends Node` 需要确认
- 如果是 `extends Node` 的 autoload，`static func` 不需要改动；如果是普通类，保持实例方法
- `DEFAULT_HERO_SPRITE` / `DEFAULT_ENEMY_SPRITE` 如果不存在资源文件，先用 `ColorRect` fallback，但路径先写好占位

### Step 2：ConfigManager 的敌人配置里增加 sprite_path

**文件**：`scripts/autoload/config_manager.gd`（敌人配置部分）

如果敌人配置是动态读取 JSON/CSV/字典的，在生成 `enemy_config` 字典时自动注入 `sprite_path`：

```gdscript
func get_enemy_config(enemy_id: String) -> Dictionary:
    var cfg: Dictionary = _enemy_configs.get(enemy_id, {}).duplicate()
    if cfg.is_empty():
        return {}
    ## 自动注入精灵图路径（如果没有显式配置）
    if not cfg.has("sprite_path"):
        var eid: int = int(enemy_id)
        cfg["sprite_path"] = get_enemy_sprite_path(eid)
    return cfg
```

这样 `NodeResolver.generate_enemy_for_floor()` 返回的 `enemy` 字典自带 `sprite_path`，`run_main.gd` 不需要再 match。

### Step 4：battle_animation_panel.gd 的 _load_sprite 增强

**文件**：`scenes/run_main/battle_animation_panel.gd`

**当前**：`_load_sprite()` 加载 `SpriteFrames` 资源后设置 `autoplay = "idle"`。

**增强**：如果 `SpriteFrames` 加载失败，自动回退为 `ColorRect` 占位，避免完全空白：

```gdscript
func _load_sprite(animated_sprite: AnimatedSprite2D, path: String) -> void:
    if path.is_empty():
        ## 无路径：保持 ColorRect 占位色
        _set_placeholder_sprite(animated_sprite, animated_sprite.name.contains("Hero"))
        return
    
    var frames: Resource = load(path)
    if frames == null or not frames is SpriteFrames:
        push_warning("[BattleAnimation] 无法加载 SpriteFrames: %s，使用占位" % path)
        _set_placeholder_sprite(animated_sprite, animated_sprite.name.contains("Hero"))
        return
    
    animated_sprite.sprite_frames = frames
    animated_sprite.autoplay = "idle"
    animated_sprite.play("idle")

func _set_placeholder_sprite(animated_sprite: AnimatedSprite2D, is_hero: bool) -> void:
    ## 占位：创建一个简单的纯色 AnimatedSprite2D（1帧）
    var placeholder: SpriteFrames = SpriteFrames.new()
    var texture := GradientTexture2D.new()
    texture.gradient = Gradient.new()
    if is_hero:
        texture.gradient.colors = [Color("#4ECDC4"), Color("#2B6B5E")]  ## 蓝色系
    else:
        texture.gradient.colors = [Color("#FF6B6B"), Color("#8B2E2E")]  ## 红色系
    texture.width = 128
    texture.height = 128
    placeholder.add_animation("idle")
    placeholder.add_frame("idle", texture)
    animated_sprite.sprite_frames = placeholder
    animated_sprite.play("idle")
```

**可选**：如果占位太复杂，可以直接不处理，让 `ColorRect` 背景色显示（`hero_portrait`/`enemy_portrait` 已经存在）。

### Step 5：ConfigManager 里增加敌人配置 JSON/CSV 读取时注入 sprite_path

如果敌人配置是外部文件（如 `res://data/enemies.json`），修改读取逻辑，给每个敌人条目自动加 `sprite_path`：

```gdscript
func _load_enemy_configs() -> void:
    ## 原有加载逻辑...
    for enemy_id in _enemy_configs.keys():
        var cfg: Dictionary = _enemy_configs[enemy_id]
        var eid: int = cfg.get("id", int(enemy_id))
        cfg["sprite_path"] = get_enemy_sprite_path(eid)
```
