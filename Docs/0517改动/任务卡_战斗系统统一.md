# 任务卡：战斗系统统一为一套 Recorder 入口

## 背景

当前有两套战斗动画入口：
- `start_playback(recorder, ...)` — PVP 使用，走 BattlePlaybackRecorder 事件流
- `start_battle(battle_result)` — 爬塔使用，走摘要模拟（lerp HP + randf 生成假事件）

**问题**：
1. 修动画逻辑要改两处（跳过、Tween、HP更新、结算）
2. 爬塔角色不动：摘要模式不生成 recorder，没传 sprite_path，`_play_anim` 被空值拦截
3. PVP 回合数异常：`total_rounds` 从 `battle_result.turns_elapsed` 取，可能为 0/1，导致提前结算
4. 两套 HP 初始化逻辑：playback 从 max_hp 开始，battle 从 battle_result 读实际 HP

**目标**：所有战斗统一走 `start_playback(recorder, ...)`，删掉 `start_battle`。

## 执行步骤

### Step 1：爬塔战斗也生成 recorder

**文件**：`scripts/systems/run_controller.gd`

当前 `_run_battle_engine()` 已创建 `BattlePlaybackRecorder` 并传给 `BattleEngine`，但爬塔分支在 `_process_node_result()` 里直接构造 `battle_result` 字典传给 UI，没有把这个 recorder 带出去。

**修改**：
- `_run_battle_engine()` 已返回 `result`（含 `playback_recorder`），确认这一点不变
- `_process_node_result()` 战斗节点分支，`_last_battle_summary` 里保留 `playback_recorder`
- `_on_battle_ended()` 信号发射时，确保 `battle_result` 字典里有 `playback_recorder` key

```gdscript
# _process_node_result() 战斗节点分支（约 458-476 行）
# 当前已调用 _run_battle_engine(enemy_config_id)，其返回 result 含 playback_recorder
# 确保 _last_battle_summary 和 EventBus.emit_signal("battle_ended", ...) 传的是同一个字典

# 修改 _last_battle_summary 构造：
var battle_result = result.get("battle_result", {})
if battle_result.is_empty():
    battle_result = _run_battle_engine(enemy_config_id)
    result["battle_result"] = battle_result

# 确保 recorder 在 battle_result 里（_run_battle_engine 已保证，这里二次确认）
_last_battle_summary = {
    "playback_recorder": battle_result.get("playback_recorder", null),
    "hero_name": hero_name,
    "enemy_name": enemy_name,
    "hero_max_hp": _hero.max_hp,
    "enemy_max_hp": enemy_max_hp,
    "hero_final_hp": battle_result.get("hero_remaining_hp", _hero.current_hp),
    "enemy_final_hp": 0 if battle_result.get("winner") == "player" else enemy_max_hp,
    "rounds": battle_result.get("turns_elapsed", 1),
    "winner": battle_result.get("winner", "player"),
    "hero_sprite_path": hero_sprite_path,   # Step 2 补
    "enemy_sprite_path": enemy_sprite_path, # Step 2 补
}
EventBus.emit_signal("battle_ended", _last_battle_summary)
```

### Step 2：爬塔分支传 sprite_path

**文件**：`scenes/run_main/run_main.gd` → `_on_battle_ended()`

当前 `_on_battle_ended()` 里爬塔分支（`recorder == null or size() == 0`）走 `_show_battle_summary`，不播放动画。要改成：如果 recorder 存在且事件数 > 0，不管 PVP 还是爬塔都走 `start_playback`。

**修改**：
```gdscript
func _on_battle_ended(battle_result: Dictionary) -> void:
    _pending_battle_result = battle_result
    var recorder = battle_result.get("playback_recorder", null)
    
    if recorder != null and recorder.get_events().size() > 0:
        var hero_data = battle_result.get("hero", {})
        var enemy_data = battle_result.get("enemies", [{}])[0]
        var hero_name = hero_data.get("name", "英雄")
        var enemy_name = enemy_data.get("name", "敌人")
        var hero_max_hp = hero_data.get("max_hp", 100)
        var enemy_max_hp = enemy_data.get("max_hp", 100)
        var turns_elapsed = battle_result.get("turns_elapsed", 0)
        
        # 补 sprite_path
        var hero_sprite_path: String = ""
        var summary = _run_controller.get_current_run_summary()
        var runtime_hero = summary.get("hero", {})
        var hero_config_id = runtime_hero.get("hero_config_id", 0)
        match hero_config_id:
            1: hero_sprite_path = "res://assets/characters/warrior/hero_frames.tres"
            2: hero_sprite_path = "res://assets/characters/shinobi/hero_frames.tres"
            # 后续英雄继续补
        
        var enemy_cfg_id = enemy_data.get("config_id", 2001)
        var enemy_sprite_path = "res://assets/characters/enemies/enemy_%d_frames.tres" % enemy_cfg_id
        # 或者用 enemy_data.get("sprite_path", "")
        
        battle_animation_panel.reset_panel()
        _show_modal_panel(battle_animation_panel)
        
        # 统一走 start_playback
        battle_animation_panel.start_playback(
            recorder,
            hero_name, enemy_name,
            hero_max_hp, enemy_max_hp,
            [], [],  # partners 暂空
            turns_elapsed,
            hero_sprite_path,
            enemy_sprite_path
        )
        
        if not battle_animation_panel.confirmed.is_connected(_on_battle_animation_finished):
            battle_animation_panel.confirmed.connect(_on_battle_animation_finished, CONNECT_ONE_SHOT)
    else:
        # 真的没有 recorder，才直接结算
        _show_battle_summary(battle_result)
```

**注意**：删除原 `_on_combat_confirmed()` 里 `battle_animation_panel.reset_panel()` + `_show_modal_panel()` + `_run_controller.select_node()` 的"先开空面板再执行战斗"逻辑。改为：
- `_on_combat_confirmed()` 只调 `_run_controller.select_node(_combat_selected_index)`
- 战斗执行完毕后 `_on_battle_ended()` 统一接管面板显示和动画播放

### Step 3：删掉 `start_battle` 入口

**文件**：`scenes/run_main/battle_animation_panel.gd`

**操作**：
1. 删除整个 `start_battle(battle_result)` 函数（约 196-249 行）
2. 删除 `_generate_simulated_turn()`（约 429-496 行）—— 不再需要模拟事件
3. 删除 `_sim_total_rounds`、`_sim_hero_final_hp`、`_sim_enemy_final_hp`、`_sim_victory` 变量 —— 这些只有摘要模式用
4. `_play_next_turn()` 里 `else` 分支（`_generate_simulated_turn()`）改为：如果 recorder 没有当前回合事件，直接 `turn_timer.start(1.0)` 等待下一回合，不生成假事件

```gdscript
# _play_next_turn() 修改后：
if _recorder != null and _events_by_turn.has(_current_round):
    var events: Array = _events_by_turn[_current_round]
    if events.size() > 0:
        if _event_tween != null and _event_tween.is_valid():
            _event_tween.kill()
        _event_tween = create_tween()
        for i in range(events.size()):
            _event_tween.tween_callback(_safe_process_event.bind(events[i]))
            _event_tween.tween_callback(_update_hp_display)
            _event_tween.tween_interval(0.5)
        _event_tween.tween_callback(func(): turn_timer.start(1.0))
    else:
        turn_timer.start(1.0)
else:
    # 无事件，等 timer 推进
    turn_timer.start(1.0)
```

### Step 4：`start_playback` 回合数修复

**文件**：`scenes/run_main/battle_animation_panel.gd`

**当前 bug**：`total_rounds` 参数从外部传入，可能为 0/1，导致提前结算。

**修改**：`start_playback()` 里 `_sim_total_rounds` 改用 recorder 实际最大回合数，忽略传入参数：

```gdscript
# start_playback() 里：
_events_by_turn = {}
_turn_keys = []
var _real_max_turn: int = 0

if _recorder != null and _recorder.has_method("get_events_by_turn"):
    var raw_events: Dictionary = _recorder.get_events_by_turn()
    for k in raw_events.keys():
        var ik: int = int(k)
        _events_by_turn[ik] = raw_events[k]
        _turn_keys.append(ik)
        _real_max_turn = maxi(_real_max_turn, ik)
    _turn_keys.sort()

# 关键修改：用 recorder 真实回合数，不用传入的 total_rounds
_sim_total_rounds = maxi(_real_max_turn, 1)
```

传入参数 `total_rounds` 可保留但不再用于 `_sim_total_rounds`（或删掉）。

### Step 5：统一 HP 初始化

**文件**：`scenes/run_main/battle_animation_panel.gd`

**当前问题**：`start_playback()` 把 `_hero_hp` 和 `_enemy_hp` 都初始化为 `maxi(1, hero_max_hp)` 即满血。但爬塔战斗开场时英雄可能不是满血（之前受过伤）。

**修改**：`start_playback()` 接收 `hero_start_hp` 和 `enemy_start_hp` 参数，从满血改为实际开场 HP：

```gdscript
# start_playback() 签名增加两个参数：
func start_playback(recorder, hero_name: String, enemy_name: String,
                    hero_max_hp: int, enemy_max_hp: int,
                    _hero_partners: Array, _enemy_partners: Array,
                    total_rounds: int = 0,
                    hero_start_hp: int = -1,    # 新增
                    enemy_start_hp: int = -1,   # 新增
                    hero_sprite_path: String = "",
                    enemy_sprite_path: String = "") -> void:
    # ...
    _hero_max_hp = maxi(1, hero_max_hp)
    _hero_hp = hero_start_hp if hero_start_hp >= 0 else _hero_max_hp
    _enemy_max_hp = maxi(1, enemy_max_hp)
    _enemy_hp = enemy_start_hp if enemy_start_hp >= 0 else _enemy_max_hp
```

`run_main.gd` 调用时传入实际 HP：
```gdscript
battle_animation_panel.start_playback(
    recorder, hero_name, enemy_name,
    hero_max_hp, enemy_max_hp,
    [], [],
    turns_elapsed,
    hero_data.get("hp", hero_max_hp),      # 实际开场 HP
    enemy_data.get("hp", enemy_max_hp),    # 实际开场 HP（满血）
    hero_sprite_path,
    enemy_sprite_path
)
```

### Step 7：清理 `_on_combat_confirmed` 调用链

**文件**：`scenes/run_main/run_main.gd`

**当前**：`_on_combat_confirmed()` 先 `reset_panel()` + `_show_modal_panel()` 开空面板，再 `_run_controller.select_node()` 执行战斗。`select_node()` 执行完后 `_on_battle_ended()` 信号到达，此时面板已开着，直接往里灌数据。

**修改后**：`_on_combat_confirmed()` 只负责两件事：
1. 隐藏 CombatConfirmPanel
2. 调 `_run_controller.select_node()`

战斗面板由 `_on_battle_ended()` 统一打开：
```gdscript
func _on_combat_confirmed() -> void:
    AudioManager.play_ui("confirm")
    combat_confirm_panel.visible = false
    enemy_info_panel.visible = false
    battle_summary_panel.visible = false
    
    if _combat_selected_index >= 0:
        _run_controller.select_node(_combat_selected_index)
        _combat_selected_index = -1
```
