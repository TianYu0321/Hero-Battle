# 任务卡：清理默认界面怪物信息残留逻辑

## 背景

**设计意图**：默认界面（4选项按钮展示时）不承载任何怪物信息。怪物信息只在"战斗被聚焦"时以剪影/浮层形式出现：
1. 点击"战斗"按钮 → 弹出 `CombatConfirmPanel`（半透明剪影预览）
2. 点击"进入战斗" → 进入 `BattleAnimationPanel`（完整战斗UI）

**当前残留**：
- `run_main.gd` 里有 `_update_monster_info(node_options)` 函数（约 762-808 行），会遍历节点选项查找战斗节点，然后在默认界面的 `EnemyInfoPanel` 上显示敌人名称、HP、预计损失血量
- `_on_enemy_encountered(enemy_data)` 信号回调调用了 `update_enemy_info(enemy_data)`，也会直接显示敌人信息
- `_on_node_options_presented()` 末尾虽然有 `enemy_info_panel.visible = false`，但如果信号驱动链中有其他代码调用了 `update_enemy_info` 或 `_update_monster_info`，默认界面还是会闪现/残留怪物信息

**目标**：彻底清理默认界面的怪物信息展示逻辑，只保留 `CombatConfirmPanel` 和 `BattleAnimationPanel` 中的聚焦式展示。

### Step 3：改造 _on_enemy_encountered 信号处理

**文件**：`scenes/run_main/run_main.gd`

**当前代码**（约 747-760 行）：
```gdscript
func _on_enemy_encountered(enemy_data: Dictionary) -> void:
    update_enemy_info(enemy_data)

func update_enemy_info(enemy_data: Dictionary) -> void:
    enemy_info_panel.visible = true
    enemy_name_label.text = "敌人: %s" % enemy_data.get("name", "???")
    var max_hp: int = enemy_data.get("max_hp", 0)
    var current_hp: int = enemy_data.get("current_hp", max_hp)
    enemy_hp_label.text = "HP: %d/%d" % [current_hp, max_hp]
    var estimated_damage: int = enemy_data.get("estimated_damage", 0)
    predicted_damage_label.text = "预计损失血量: %d" % estimated_damage
```

**问题**：`NodeResolver._resolve_battle()` 里 `EventBus.emit_signal("enemy_encountered", enemy)` 会在战斗节点执行时触发，导致默认界面直接显示敌人信息。

**修改方案**：

方案 A（推荐）：`update_enemy_info()` 不再直接显示面板，只缓存数据到临时变量。`CombatConfirmPanel` 需要时再读取。

```gdscript
## 缓存当前敌人数据（供 CombatConfirmPanel 使用）
var _cached_enemy_data: Dictionary = {}

func _on_enemy_encountered(enemy_data: Dictionary) -> void:
    _cached_enemy_data = enemy_data.duplicate()
    ## 不直接显示 enemy_info_panel！
    print("[RunMain] 缓存敌人数据: %s" % enemy_data.get("name", "???"))

func update_enemy_info(enemy_data: Dictionary) -> void:
    ## 此函数保留供 CombatConfirmPanel 显式调用
    enemy_info_panel.visible = true
    enemy_name_label.text = "敌人: %s" % enemy_data.get("name", "???")
    var max_hp: int = enemy_data.get("max_hp", 0)
    var current_hp: int = enemy_data.get("current_hp", max_hp)
    enemy_hp_label.text = "HP: %d/%d" % [current_hp, max_hp]
    var estimated_damage: int = enemy_data.get("estimated_damage", 0)
    predicted_damage_label.text = "预计损失血量: %d" % estimated_damage
```

然后在 `_show_combat_preview()` 中使用缓存数据：
```gdscript
func _show_combat_preview(opt: Dictionary, index: int) -> void:
    _combat_selected_index = index
    option_container.visible = false
    
    ## 使用缓存的敌人数据或从 opt 读取
    var enemy_cfg: Dictionary = opt.get("enemy_config", {})
    if enemy_cfg.is_empty() and not _cached_enemy_data.is_empty():
        enemy_cfg = _cached_enemy_data
    
    if not enemy_cfg.is_empty():
        enemy_info_panel.visible = true
        enemy_info_panel.modulate = Color(1, 1, 1, 0.6)
        ## 复用 update_enemy_info 显示
        update_enemy_info(enemy_cfg)
        ## 覆盖为剪影模式（隐藏具体 HP，显示 ???）
        enemy_hp_label.text = "HP: ???"
        predicted_damage_label.text = ""
        risk_label.text = ""
    else:
        enemy_info_panel.visible = false
    
    combat_confirm_panel.visible = true
```

方案 B（更激进）：删除 `enemy_encountered` 信号和 `update_enemy_info()`，完全由 `_show_combat_preview()` 自行从 `opt.get("enemy_config")` 读取显示。如果 `NodeResolver` 发射 `enemy_encountered` 只是为了让 UI 预加载数据，可以改为不发射，或改为 `enemy_data_cached` 信号。
